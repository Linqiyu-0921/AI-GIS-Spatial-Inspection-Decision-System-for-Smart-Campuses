# 01 移动端原型设计说明 — 日常巡检场景

> 基于设计系统规范与日常巡检PRD流程，定义移动端（巡检员端）5个核心页面的原型结构、关键元素与交互逻辑。所有尺寸、颜色、字号引用自 `00-design-system.md`，流程对应 `04-scenario-daily-inspection.md` 步骤①~⑧，用户情绪与痛点参考 `02-journey-maps.md` 地图一。

---

## 页面总览

| 序号 | 页面名称 | 对应PRD步骤 | 核心目标 | 情绪设计重点 |
|------|---------|------------|---------|-------------|
| 1 | 任务中心首页 | ①计划接收 | 让巡检员一眼看清今日任务总量与紧急事项 | 降低新手焦虑（预估时长+难度提示），减少老手枯燥感（智能调频提示） |
| 2 | 巡查执行页 | ②执行 | 支撑户外移动场景下的导航、打卡、清单执行 | 确保"不离开地图看数据"，弱化老手的被约束感，强化新手的到位确认 |
| 3 | AI拍照识别页 | ③发现+④分支判断 | 将问题发现与AI判断压缩到最短路径 | 拍照后1秒内给出结果，消灭"不确定感"和"描述焦虑" |
| 4 | 问题上报页 | ③发现（提交） | 让巡检员只需"确认"而非"填写" | AI预填所有字段，语音补充代替打字，位置自动绑定 |
| 5 | 巡检完成页 | ⑧分析（个人视角） | 给巡检员即时闭环反馈，确认工作价值 | 消灭"石沉大海"感，日报可分享增强成就感 |

## 页面一：任务中心首页

### 1.1 页面名称与用途

- **页面名称**：任务中心首页（Mobile Home）
- **用途**：巡检员每日打开APP的第一界面，聚合当日巡检计划、待办工单、紧急推送三大信息源，作为全部工作的统一入口。
- **对应PRD步骤**：①计划 — 接收任务
- **情绪设计**：新手看到"预估时长2.5h·难度普通"降低焦虑；老手看到"A区已智能降频"感到经验被尊重。

### 1.2 布局结构

顶部 1/5 区域：状态栏+日期天气+今日任务统计  
中部 3/5 区域：巡检计划列表（可滚动）  
底部 1/5 区域：待办工单快捷入口 + 紧急推送浮条

> 整体为单列纵向流式布局，左右安全边距 16px，模块间距 16px，背景色 `#F3F4F6`。

### 1.3 关键元素清单

| 元素名称 | 位置 | 内容/状态 | 组件来源（设计系统） |
|---------|------|----------|-------------------|
| 日期标签 | 顶部左对齐 | "4月27日 周一"，H3 16px/500，文字主色 | 3.1 移动端字号 |
| 天气微标 | 日期右侧 | 图标+"多云 18°C"，辅助文字 12px/400，文字次色 | 7.2 图标尺寸（小16px） |
| 任务统计卡 | 顶部下方 | 白色卡片，圆角12px，三列等宽：待巡检/已完成/异常数，数据大字 32px/700 + 辅助文字标签 | 5.3 信息卡片 + 3.1 数据大字 |
| 计划列表标题 | 中部顶端 | "今日巡检计划（3条路线）"，H2 18px/600 | 3.1 移动端字号 |
| 巡检计划卡片 | 中部主体 | 白色卡片，圆角12px，左高4px色条区分状态（待执行-警告黄/进行中-科技蓝/已完成-成功绿） | 5.3 工单卡片 |
| 卡片内容 | 卡片内部 | 路线名称（H3）+ 园区楼栋范围（正文）+ 点位数/预计时长（辅助文字）+ 状态标签 | 5.4 状态标签 |
| 快捷工单入口 | 底部上方 | 横向滚动的缩略卡片，显示待处理工单数+最近一条工单标题 | 5.3 信息卡片（紧凑型） |
| 紧急推送浮条 | 底部固定 | 红色底色条（错误红 `#EF4444`），白字，显示紧急事件数量，点击展开 | 2.2 功能色-错误红 |
| 底部导航 | 最底部 | 4 Tab：任务中心/地图/消息/我的，图标24px+文字10px，激活态科技蓝-500 | 5.5 底部导航 |

### 1.4 交互逻辑

| 用户操作 | 系统反馈 | 备注 |
|---------|---------|------|
| 下拉页面 | 触发刷新动画，同步最新计划与工单状态 | 符合移动端刷新心智模型 |
| 点击巡检计划卡片 | 跳转至「巡查执行页」，带入该路线ID、点位列表、已巡进度 | 核心入口，操作路径=1步 |
| 左滑计划卡片 | 露出快捷操作：「开始巡检」（蓝色）/「跳过今日」（灰色） | 5.8 任务列表左滑操作 |
| 点击快捷工单入口 | 跳转至「工单详情页」（本文档范围外） | 减少查找路径 |
| 点击紧急推送浮条 | 展开全屏紧急事件列表，或跳转应急事件专用页 | P0设计行动项：紧急按钮常驻 |
| 切换底部Tab | 切换至对应主页面 | 底部导航标准行为 |

### 1.5 AI/GIS嵌入点标注

| 嵌入点 | 位置 | 说明 |
|--------|------|------|
| **AI-预测分析** | 计划卡片右上角 | 显示"智能调频"微标（如"A区已降频"），点击弹出说明浮层 |
| **AI-预估时长** | 计划卡片内 | 基于历史该路线平均耗时计算"预计2.5h"，新手可见，老手可折叠 |
| **GIS-路线预览** | 计划卡片右侧 | 缩略地图（80×80px圆角矩形），展示路线起止与点位分布，点击放大 |

## 页面二：巡查执行页

### 2.1 页面名称与用途

- **页面名称**：巡查执行页（Inspection Execution）
- **用途**：巡检员在现场移动执行的核心工作界面，融合GIS地图、检查清单、打卡操作三大模块，确保"地图即主界面"。
- **对应PRD步骤**：②执行
- **情绪设计**：新手靠GPS+围栏确认"到没到"，弱化焦虑；老手可切换「熟悉路线模式」弱化强制导航。

### 2.2 布局结构

顶部 1/2 区域：GIS地图（全宽，高度占视口50%）  
中部 1/6 区域：检查清单进度条 + 当前点位信息卡  
底部 1/3 区域：打卡按钮 + 拍照上报按钮 + 底部操作栏

> 地图区域允许全屏手势操作（缩放/拖拽），其他UI元素悬浮叠加于地图之上。背景色 `#FFFFFF`。

### 2.3 关键元素清单

| 元素名称 | 位置 | 内容/状态 | 组件来源（设计系统） |
|---------|------|----------|-------------------|
| 地图容器 | 顶部大半屏 | GIS地图，显示当前位置脉冲圆环、巡检路线（已巡实线/规划虚线）、点位标记 | 5.6 地图组件 |
| 定位点 | 地图中央偏下 | 科技蓝-500脉冲动画圆环，标识当前GPS位置 | 5.6 定位点 |
| POI标记点 | 地图各点位 | 水滴形容器，功能色区分：待检-警告黄/已检-成功绿/异常-错误红 | 5.6 标记点 |
| 地图工具栏 | 地图右上角垂直排列 | 定位（回当前位置）、缩放+/-、图层切换，图标按钮 44×44px | 5.1 图标按钮 |
| 进度条 | 地图下方 | 科技蓝-400填充轨道，显示"已完成 12/25 点位"+百分比 | 5.7 进度条 |
| 当前点位卡 | 进度条下方 | 白色卡片，显示当前点位名称+所属区域+checklist完成数 | 5.3 信息卡片 |
| 打卡按钮 | 底部左侧 | 主按钮 44px高，科技蓝-500填充，文字"确认到位打卡"，围栏内可用/围栏外置灰 | 5.1 主按钮 |
| 拍照上报FAB | 底部右侧 | 悬浮按钮（FAB），圆形 56px，错误红填充，相机图标白色 | 5.1 悬浮按钮（FAB） |
|  checklist浮层 | 底部可上滑 | 上滑抽屉，显示当前点位所有checklist项，逐项勾选框 | 5.8 列表-任务列表 |
| 导航提示条 | 地图顶部 | 半透明白色条，显示"下一站：B座3F消防通道，预计步行3min" | 自定义，文字次色 |

### 2.4 交互逻辑

| 用户操作 | 系统反馈 | 备注 |
|---------|---------|------|
| 地图缩放/拖拽 | 地图响应手势，POI标记点动态聚合/展开，轨迹线随缩放调整粗细 | 5.6 聚合点规则 |
| 点击POI标记点 | 弹出地图弹窗卡片，显示点位名称、状态、历史问题数 | 5.3 地图弹窗卡片 |
| 进入围栏（≤30m） | 打卡按钮从置灰（背景按压 `#E5E7EB`）变为可用（科技蓝-500），伴随轻微震动反馈 | 硬件震动API，操作路径=1步 |
| 点击打卡按钮 | 记录GPS+时间戳，该点位标记变绿，自动展开checklist浮层，语音播报"已打卡，请完成检查清单" | [假设] 语音播报可关闭 |
| checklist项勾选 | 勾选框变蓝，已完成项可折叠；全部勾选后点位卡显示"已完成"标签 | 实时反馈降低遗忘焦虑 |
| 点击拍照上报FAB | 跳转至「AI拍照识别页」 | 紧急问题随时上报，不中断巡检流程 |
| 底部抽屉上滑 | 展开完整checklist列表，下滑收起 | 地图与清单不割裂 |
| 偏离路线>50m | 顶部导航提示条变橙，语音提醒"您已偏离路线，请返回" | PRD AI嵌入-实时预警 |
| 连续跳过2项checklist | 弹窗提醒"请逐项完成检查，快速完成将被记录" | PRD AI嵌入-防作弊 |

### 2.5 AI/GIS嵌入点标注

| 嵌入点 | 位置 | 说明 |
|--------|------|------|
| **GIS-轨迹记录** | 地图底层 | GPS每10秒采样，实时绘制蓝色轨迹线；管理层同步可见 |
| **GIS-路线导航** | 地图叠加层 | 虚线规划路线+箭头方向标，引导至下一待检点位 |
| **GIS-围栏打卡** | 点位周围 | 蓝色缓冲区（`rgba(37,99,235,0.15)`），进入30m范围触发打卡可用态 |
| **AI-异常区域提示** | 接近高频问题区时 | 顶部导航条变警告黄，显示"该区域近7天出现3次照明故障，请重点关注" |
| **AI-实时预警** | 偏离/跳项时 | 弹窗或顶部条提示，减少管理层事后追责 |
| **GIS-热力图切换** | 地图工具栏-图层 | 可切换显示问题热力图（低/中/高三色），辅助巡检员关注重点 |

## 页面三：AI拍照识别页

### 3.1 页面名称与用途

- **页面名称**：AI拍照识别页（AI Photo Recognition）
- **用途**：巡检员发现问题后拍照，AI在1秒内返回识别结果（问题类型、置信度、建议等级），将"发现→分类→判断"三步压缩为"拍照→确认"一步。
- **对应PRD步骤**：③发现 + ④分支判断
- **情绪设计**：消灭新手"不知道报什么类型"和"不确定够不够严重"的焦虑，AI给结果只需确认。

### 3.2 布局结构

顶部 9/10 区域：相机取景框（全屏或大半屏）  
底部 1/10 区域：拍照快门按钮  
拍照后：识别结果浮层从底部滑入，覆盖底部1/3~1/2区域

> 取景框全屏最大化视野，拍照后不过渡跳转，直接在当前页叠加结果浮层，保持上下文连续。

### 3.3 关键元素清单

| 元素名称 | 位置 | 内容/状态 | 组件来源（设计系统） |
|---------|------|----------|-------------------|
| 取景框 | 全屏背景 | 相机实时预览，4:3或全屏比例，中央有半透明对焦框 | 系统相机API |
| 问题类型快捷标签 | 取景框底部（拍照前） | 横向滚动的历史常用标签（如"堆物/照明/破损"），点击直接带入选项 | 5.4 类型标签 |
| 快门按钮 | 底部中央 | 圆形按钮 72px，白色外环+科技蓝-500内芯，底部安全区域适配 | 自定义，大于最小点击区 |
| 相册入口 | 快门左侧 | 图标按钮，调取最近照片缩略图 | 5.1 图标按钮 |
| 切换摄像头 | 快门右侧 | 图标按钮，前/后置切换 | 5.1 图标按钮 |
| 识别结果浮层 | 拍照后底部滑入 | 白色浮层，圆角顶部16px，高度占屏1/3~1/2 | 5.3 信息卡片变体 |
| AI识别标签 | 浮层顶部 | "AI识别：消防通道堆物"，H2 18px/600，科技蓝-500前缀"AI"角标 | 3.1 H2 + 5.4 标签 |
| 置信度条 | 标签下方 | 进度条样式，显示"置信度 92%"，≥85%为科技蓝，<85%为警告黄 | 5.7 进度条 + 功能色 |
| 建议分级 | 置信度下方 | "建议分级：P2·安全隐患"，优先级标签（橙色P2）+ 类型说明 | 5.4 优先级标签 |
| 判定依据 | 建议下方 | 辅助文字 12px，"历史同类案例：近30天该类问题3次，均走工单流程" | 3.1 辅助文字 |
| 操作按钮组 | 浮层底部 | 主按钮"确认并上报"+次按钮"重拍"+文字按钮"手动修正" | 5.1 按钮组合 |
| 手动选择面板 | 点击手动修正后 | 底部弹出选择器，问题类型一级分类+二级细分 | 5.2 选择器 |

### 3.4 交互逻辑

| 用户操作 | 系统反馈 | 备注 |
|---------|---------|------|
| 点击快门 | 触发拍照，屏幕闪白+快门音效，1秒内显示识别结果浮层 | 1秒SLA，超时显示"识别中..."骨架屏 |
| 置信度≥85% | 问题类型自动填充，建议分级高亮，主按钮为"确认并上报" | AI可信时减少用户操作 |
| 置信度<85% | 置信度条变警告黄，提示"识别不确定，请手动选择或重拍"，主按钮变为"手动选择问题类型" | 不给错误确定性 |
| 点击"确认并上报" | 携带识别结果跳转至「问题上报页」，所有字段预填充 | 操作路径连续 |
| 点击"重拍" | 浮层下滑收起，回到取景框状态 | 保留拍照上下文 |
| 点击"手动修正" | 唤起底部选择器，覆盖问题类型/严重等级，用户选择后实时更新浮层 | 用户始终有最终控制权 |
| 多指捏合取景框 | 相机变焦（如有支持） | 标准相机行为 |

### 3.5 AI/GIS嵌入点标注

| 嵌入点 | 位置 | 说明 |
|--------|------|------|
| **AI-图像识别** | 拍照后触发 | 云端/边缘模型识别问题类型，返回Top3候选+置信度；目标识别准确率影响用户信任 |
| **AI-严重程度预测** | 识别结果中 | 基于图像+类型+历史数据，自动判定P0-P3等级，展示判定依据和历史案例 |
| **AI-分支引导** | 建议分级旁 | 向新手展示"为什么是这个等级"的自然语言说明，如"消防通道堵塞影响疏散，属安全隐患" |
| **GIS-自动定位** | 后台静默执行 | 拍照同时记录GPS坐标，作为上报位置，精度不足时带入上一打卡点位 |

## 页面四：问题上报页

### 4.1 页面名称与用途

- **页面名称**：问题上报页（Issue Report）
- **用途**：AI预填所有结构化字段后，巡检员仅需确认信息、补充语音描述、选择严重程度，一键提交。将传统"填写表单"转化为"审核确认"。
- **对应PRD步骤**：③发现 — 提交
- **情绪设计**：消灭"手写记录再回办公室录入"的双倍工作感，语音代替打字适合户外场景，自动定位消灭"我在哪"的困惑。

### 4.2 布局结构

顶部 1/4 区域：AI自动填单区（问题类型/位置/描述）  
中部 1/3 区域：照片缩略图+语音输入按钮  
底部 1/3 区域：严重程度选择+提交按钮

> 单列纵向布局，卡片式分组，组间距16px，背景色 `#F3F4F6`。表单行高48px。

### 4.3 关键元素清单

| 元素名称 | 位置 | 内容/状态 | 组件来源（设计系统） |
|---------|------|----------|-------------------|
| 页面标题栏 | 顶部 | "问题上报"，H1 20px/600，左侧返回箭头，右侧"保存草稿"文字按钮 | 3.1 H1 + 5.1 文字按钮 |
| AI预填提示条 | 标题下方 | 浅蓝底色条（科技蓝-100 `#DBEAFE`），"AI已自动填充以下信息，请核对" | 2.3 中性色/主色浅色 |
| 问题类型字段 | 预填区第一行 | 标签"问题类型"+值"消防通道堆物"+右侧编辑图标；值高亮科技蓝-500 | 5.2 选择器（只读态） |

### 7.4 交互逻辑

| 用户操作 | 系统反馈 | 备注 |
|---------|---------|------|
| 进入页面 | 根据工单状态展示对应界面，状态标签高亮 | 不同状态可见元素不同 |
| 状态变更 | 收到推送时，状态标签切换+Toast"工单状态更新：处置中" | PRD SLA：状态变更实时推送 |
| 点击进度照片 | 进入大图预览，支持左右滑动查看全部 | 标准图片预览 |
| 点击"查看实时位置" | 底部滑出地图浮层，显示处置人实时位置+工单位置 | GIS实时定位 |
| 点击验收拍照 | 唤起相机，拍摄后缩略图显示，验收按钮变为可用 | 验收必须拍照留证 |
| 点击"验收通过" | 弹窗确认"确认验收通过？"，确认后状态变"已闭环"，Toast提示 | 闭环最后一步 |
| 点击"重新分派" | 弹窗选择重新分派原因（处置不到位/未按时到场/其他），提交后工单回退至待处置 | 质检不通过时启用 |
| 点击已闭环的问题卡片 | 展示完整处置时间线+验收照片 | 历史可追溯 |
