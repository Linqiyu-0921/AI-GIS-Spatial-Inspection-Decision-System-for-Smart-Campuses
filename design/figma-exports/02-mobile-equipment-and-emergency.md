# 移动端原型设计说明 — 设备检查 + 应急场景

> 覆盖设备专项检查（2 页）与应急事件响应（3 页）共 5 个移动端页面，延续设计系统 `00-design-system.md` 的视觉基因，适配技术员与巡检员在户内/应急场景下的操作需求。

---

## 页面 1：设备检查清单页

### 1.1 页面名称和用途

- **名称**：设备检查清单页（Equipment Checklist）
- **用途**：技术员到达设备点位后，按系统预置的检查清单逐项检查、拍照留证、记录结果，并提交检查记录。是设备专项检查流程的核心执行页面。

### 1.2 布局结构

```
┌─────────────────────────────┐  ← 顶部导航栏（48px）
│  ←  设备检查清单              │
├─────────────────────────────┤
│ ┌─────────────────────────┐ │  ← 设备信息卡片
│ │ 电梯 #EL-03-06           │ │
│ │ 月检 · B区3F · 预计30min │ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ 检查进度 3/8 ▓▓▓░░░░░       │  ← 进度条
├─────────────────────────────┤
│ ┌─────────────────────────┐ │  ← 检查项 1（已展开）
│ │ ■ 运行平稳性            │ │
│ │   启停平稳，无异常抖动   │ │
│ │   [拍照] [录音] [备注]   │ │
│ │   ✓ 正常               │ │
│ └─────────────────────────┘ │
│ ┌─────────────────────────┐ │  ← 检查项 2（已展开+异常）
│ │ ■ 门机系统              │ │
│ │   开关门顺畅，防夹有效   │ │
│ │   [拍照] [录音] [备注]   │ │
│ │   ✗ 异常  标红高亮       │ │
│ │   ┌───────────────────┐ │ │
│ │   │ AI识别结果        │ │ │  ← AI辅助区域
│ │   │ 外观照片分析：     │ │ │
│ │   │ · 锈蚀 置信度 92% │ │ │
│ │   │ · 变形 置信度 67% │ │ │
│ │   └───────────────────┘ │ │
│ └─────────────────────────┘ │
│ ┌─────────────────────────┐ │  ← 检查项 3（收起态）
│ │ □ 安全装置  ·  待检查   │ │
│ └─────────────────────────┘ │
│ ...                         │
├─────────────────────────────┤
│ ┌─────────────────────────┐ │  ← 底部固定操作栏（+ 安全间距）
│ │      [提交检查记录]      │ │     主按钮（科技蓝-500，44px高）
│ └─────────────────────────┘ │
└─────────────────────────────┘
```

### 1.3 关键元素清单

| 元素 | 样式 | 说明 |
|------|------|------|
| 顶部导航栏 | 高度 48px，左侧返回箭头 + 标题"设备检查清单"（H1，20px/600） | 全局统一 |
| 设备信息卡片 | 白色背景，圆角 12px，内边距 16px，阴影 Shadow-S | 展示设备名称、编号、检查类型标签、位置、预计时长 |
| 检查类型标签 | 状态标签样式，科技蓝-100 底色 + 科技蓝-500 文字 | 如"月检""周检""季检" |
| 进度条 | 科技蓝-500 填充，轨道 `#E5E7EB`，高度 4px，圆角 2px | 实时显示"已完成项/总项" |
| 检查项卡片 | 白色背景，圆角 12px，内边距 16px，左侧 4px 色条（正常=成功绿，异常=错误红，待检=警告黄） | 每项一个卡片 |
| 检查项标题 | H3，16px/500 | 检查项目名称 |
| 检查标准说明 | 正文，14px/400，文字次色 `#6B7280` | 检查标准简述 |
| 操作按钮组 | 图标按钮 44×44px（拍照📷、录音🎤、备注📝） | 每项独立操作 |
| 结果单选 | 横向排列："正常"（成功绿）/ "异常"（错误红） | 二选一，必选 |
| AI辅助区域 | 科技蓝-50 底色（`#EFF6FF`），圆角 8px，内边距 12px | 仅异常项或拍照后展示 |
| OCR读数展示 | 数据大字 32px/700 + 单位 + 标准范围对比 | 表盘拍照后自动识别 |
| 底部提交按钮 | 主按钮，全宽，高度 44px，圆角 8px | 全部项完成后可点击 |

### 1.4 交互逻辑

| 交互 | 行为 | 状态变化 |
|------|------|---------|
| 点击检查项卡片 | 展开/收起该项详情 | 展开显示操作按钮组 + 结果选择；收起仅显示标题和状态 |
| 点击"拍照" | 唤起相机，拍摄后缩略图预览 | 照片缩略图显示在按钮下方，支持点击查看大图/删除 |
| 点击"录音" | 按住录音，松开发送，波形动画 | 录音时长显示，支持播放回放 |
| 点击"备注" | 唤起底部多行文本输入面板 | 输入框获得焦点，支持语音转文字 |
| 选择"异常" | 该项左侧色条变红，AI辅助区域自动展开 | 必填：异常描述/照片/录音至少一项 |
| 拍摄表盘照片 | AI自动 OCR 识别读数，弹出识别结果 | 展示读数 + 与标准范围比对（正常绿/超标红） |
| 拍摄外观照片 | AI自动识别缺陷类型 | 展示识别结果列表（锈蚀/泄漏/变形/裂纹/磨损 + 置信度） |
| 点击"提交" | 校验：所有项必须有结果；异常项必须有附件 | 未达标项标红提示，达标后进入提交确认弹窗 |
| 提交确认 | 弹窗展示检查摘要（正常X项/异常Y项） | 确认后提交，跳转"提交成功"页 |

### 1.5 AI / GIS 嵌入点标注

| 嵌入点 | 能力 | 触发条件 | 展示形式 |
|--------|------|---------|---------|
| **AI-外观识别** | 识别锈蚀/泄漏/变形/裂纹/磨损 | 拍摄设备外观照片后自动触发 | 在检查项下方展开 AI 辅助区域，列出识别类型 + 置信度进度条 |
| **AI-OCR表盘读数** | 识别温度/压力/电压表盘数值 | 拍摄表盘照片后自动触发 | 大字展示读数，与标准范围比对，超标标红 |
| **AI-声音分析** | 分析异常频率，判断异响类型 | 上传录音后自动触发 | 在备注区下方展示分析结果标签（如"机械摩擦""放电声"） |
| **AI-综合评分** | 综合外观+参数+声音给出健康度评分 | 提交前自动计算 | 在提交确认弹窗展示设备健康度评分（0-100） |
| **GIS-BIM楼层定位** | 展示设备所在楼层位置 | 页面加载时自动定位 | 设备信息卡片右侧展示楼层缩略图（如"B3"图标） |
| **GIS-到位验证** | 蓝牙信标/二维码验证已到场 | 点击"开始检查"时触发 | 验证通过后方可进入清单；未通过提示"请到达设备附近" |

### 2.5 AI / GIS 嵌入点标注

| 嵌入点 | 能力 | 触发条件 | 展示形式 |
|--------|------|---------|---------|
| **GIS-BIM楼层定位** | 展示设备在楼层平面图中的精确位置 | 页面加载时自动定位 | 平面图上设备位置用水滴形标记点，当前楼层高亮 |
| **GIS-设备导航** | 规划室内导航路径 | 点击"导航到设备"（可选入口） | 在 BIM 图上绘制导航路径线（科技蓝-500 实线） |

| **AI-劣化预测** | 基于历史 OCR 读数趋势预测劣化 | 有连续 3 次以上检查记录时触发 | 在历史记录模块顶部展示预警标签（如"钢丝绳磨损加速"） |

## 页面 3：紧急告警推送页

### 3.1 页面名称和用途

- **名称**：紧急告警推送页（Emergency Alert）
- **用途**：应急事件触发后，向最近可用人员（巡检员/安保/技术员）推送告警信息，要求其在 3 分钟内确认收到，并提供 IoT 数据、导航到场、联系指挥中心等快捷操作。是应急响应流程的第一步。

### 3.2 布局结构

```
┌─────────────────────────────┐
│ ┌─────────────────────────┐ │  ← 红色告警头（全屏沉浸，无导航栏）
│ │     ⚠️ 紧急告警          │ │
│ │   火灾告警 · P0          │ │
│ │   2024-04-27 14:32:18   │ │
│ │   B区3F东侧走廊          │ │
│ │                         │ │
│ │  ⏱️ 03:00 内必须确认     │ │  ← 倒计时
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ ┌─────────────────────────┐ │  ← IoT传感器数据卡片
│ │ IoT实时数据              │ │
│ │ ┌──────┐ ┌──────┐      │ │
│ │ │ 温度  │ │ 烟雾  │      │ │
│ │ │ 68°C │ │ 高    │      │ │
│ │ │ ↑超标 │ │       │      │ │
│ │ └──────┘ └──────┘      │ │
│ │ ┌──────┐ ┌──────┐      │ │
│ │ │ 水压  │ │ CO   │      │ │
│ │ │ 0.3MP│ │ 12ppm│      │ │
│ │ │       │ │       │      │ │
│ │ └──────┘ └──────┘      │ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ ┌─────────────────────────┐ │  ← 视频联动核查（如有）
│ │ 📹 视频快照              │ │
│ │ [最近摄像头画面缩略图]   │ │
│ │ AI判断：疑似明火 置信度89%│ │
│ └─────────────────────────┘ │
├─────────────────────────────┤
│ ┌─────────────────────────┐ │  ← 底部固定操作栏
│ │ [确认收到] [导航到场] [联系]│ │
│ └─────────────────────────┘ │
└─────────────────────────────┘
```

### 3.3 关键元素清单

| 元素 | 样式 | 说明 |
|------|------|------|
| 红色告警头 | 全宽，背景错误红 `#EF4444`，文字白色，内边距 24px | 无顶部导航栏，沉浸式告警 |
| 告警类型标题 | H1，20px/600，白色 | 如"火灾告警""水管爆裂""电梯困人" |
| 优先级标签 | 大标签，白色边框 + 白色文字，圆角 4px | P0/P1，与告警类型同行 |
| 时间戳 | 辅助文字，12px/400，白色 80% 不透明度 | 精确到秒 |
| 位置信息 | H3，16px/500，白色 | 如"B区3F东侧走廊" |
| 倒计时器 | 数据大字 32px/700，白色，左侧时钟图标 | 红色背景上的醒目倒计时，从 03:00 开始递减 |
| IoT 数据卡片 | 白色背景，圆角 12px，内边距 16px | 2×2 网格展示传感器数据 |
| 传感器数值 | 数据大字 32px/700 | 正常=文字主色，超标=错误红 |
| 超标标识 | 辅助文字 12px，错误红，"↑超标" | 位于数值下方 |
| 视频快照卡片 | 白色背景，圆角 12px，内边距 16px | 展示最近摄像头画面缩略图 |
| AI 判断标签 | 状态标签，错误红底色 + 白字 | 位于视频下方，如"疑似明火 置信度89%" |
| 底部操作栏 | 固定底部，白色背景，上边框 1px `#E5E7EB`，内边距 12px 16px | 三个等宽按钮 |
| "确认收到"按钮 | 主按钮，错误红填充 + 白色文字（应急场景特殊使用） | 高优先级 CTA |
| "导航到场"按钮 | 次按钮，白色填充 + 科技蓝-500 边框和文字 | 唤起地图导航 |
| "联系指挥"按钮 | 次按钮，白色填充 + 科技蓝-500 边框和文字 | 唤起拨号/即时通讯 |

### 3.4 交互逻辑

| 交互 | 行为 | 状态变化 |
|------|------|---------|
| 页面加载 | 全屏红色告警头，倒计时从 03:00 开始 | 倒计时每秒刷新，最后 30 秒数字放大脉冲动画 |
| 倒计时 ≤1min | 进入紧急状态 | 倒计时数字改为闪烁，手机震动（支持时） |
| 倒计时结束 | 自动升级至最高级 | 弹窗提示"告警已自动升级，指挥中心将强制调度" |
| 点击"确认收到" | 记录确认时间，跳转应急指挥面板 | 按钮变为"已确认"禁用态，背景成功绿 |
| 点击"导航到场" | 唤起地图应用/GIS 内置导航 | 规划从当前位置到事件点的最优路径 |
| 点击"联系指挥" | 唤起拨号面板（指挥中心热线）+ 即时通讯入口 | 底部弹出操作菜单：电话 / 语音消息 / 文字 |
| 点击视频快照 | 全屏播放实时视频流 | 进入视频全屏页，支持截图上传确认 |

### 3.5 AI / GIS 嵌入点标注

| 嵌入点 | 能力 | 触发条件 | 展示形式 |
|--------|------|---------|---------|
| **AI-视频分析** | 分析摄像头画面，判断告警真实性 | 告警触发后自动调取最近摄像头 | 在视频快照下方展示 AI 判断结果标签 |
