# 设计系统规范

> 为「智慧园区空间巡检决策系统」定义的 Figma 设计系统，覆盖移动端（巡检员端）与 PC 端（管理层端）双端视觉规范。

---

## 一、设计原则

### 1. 效率优先
巡检员在户外移动场景操作，管理层需快速获取全局态势。所有设计决策以降低认知负荷、缩短操作路径为第一目标。核心操作 ≤3 步，高频功能直达。

### 2. 空间可视化
GIS 是系统的核心引擎，地图不是装饰而是主界面。空间信息（位置、轨迹、热力分布）必须在第一屏呈现，数据与地理空间强绑定，避免"离开地图看数据"的割裂感。

### 3. 数据驱动决策
管理层端的所有展示服务于"从数据中发现问题、预判风险"。数据状态变化必须即时可见（实时轨迹、工单状态流转），趋势与异常需用视觉层级强调。

### 4. 场景适配双端
移动端聚焦"执行"——大按钮、防误触、户外可读；PC 端聚焦"管控"——信息密度高、多窗格并行、支持数据下钻。同一信息在不同端呈现方式允许差异，但视觉基因一致。

### 6.1 移动端

| 项目 | 规范 |
|------|------|

| 最大宽度 | 430px（iPhone 14 Pro Max），超出居中留白 |
| 安全区域 | 顶部状态栏 44px（非刘海屏 20px），底部手势区 34px（有底部导航时 0） |
| 导航高度 | 顶部 48px，底部 56px |
| 内容区高度 | 视口高度 - 顶部导航 - 底部导航 |
| 地图页特殊 | 地图全屏，导航/操作按钮悬浮叠加，底部可上滑抽屉展示列表 |
| 横屏限制 | [假设] 不支持横屏，锁定竖屏 |

### 6.2 PC 端

| 项目 | 规范 |
|------|------|
| 内容最大宽度 | 1440px，超出居中 |
| 最小宽度 | 1280px，低于此宽度提示"建议使用更大屏幕" |

## 七、图标规范

### 7.1 图标风格

- **全局使用线性图标（Outlined）**，线宽 1.5px，端点圆角
- 避免面性图标，保持界面轻盈感
- 选中/激活态：填充科技蓝-500，未选中态：文字次色 `#6B7280`

### 7.2 图标尺寸

| 尺寸 | 大小 | 使用场景 |
|------|------|---------|
| 小 | 16×16px | 按钮内图标、列表行内图标、输入框前后置图标 |
| 中 | 24×24px | 底部导航、侧边栏导航、工具栏 |
| 大 | 32×32px | 空状态插图图标、功能入口图标 |

### 7.3 地图图标特殊规范

| 项目 | 规范 |
|------|------|
| POI 类型图标 | 16×16px 线性，白色背景圆形底板（24×24px） |
| 标记点图标 | 水滴形容器，内部放 16×16px 图标或缩写文字 |
| 地图工具图标 | 24×24px，白色圆形底板（40×40px），带投影 |
| 聚合点数字 | 标注/标签字号（10px），白色文字，居中对齐 |
| 轨迹方向箭头 | 沿轨迹线等距分布，12×12px，与轨迹线同色 |

## 附录：Figma 搭建建议

1. **页面结构**：建议分 3 个 Page ——「Design System」（本规范落地）、「Mobile」（移动端页面）、「PC Web」（PC 端页面）
2. **组件库命名**：采用"组件名/变体/状态"结构，如 `Button/Primary/Default`、`Button/Primary/Hover`
3. **颜色样式**：按「主色/功能色/中性色/地图色」分组建立 Color Styles
4. **文本样式**：按「移动端/PC 端」分组建立 Text Styles，命名如 `Mobile/H1/Bold`、`PC/Body/Regular`
5. **Effect Styles**：建立 3 级阴影 —— `Shadow-S`（卡片悬停）、`Shadow-M`（弹窗/抽屉）、`Shadow-L`（模态框）
6. **Grid Styles**：移动端 375px 画板用 16px 边距 + 4px 栅格；PC 端用 24px 边距 + 4px 栅格
