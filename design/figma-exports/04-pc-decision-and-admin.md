# 04 PC端原型设计说明 — 决策中心 + 系统管理

> 管理层端的两个核心页面：决策中心承载数据驱动决策的价值闭环，系统管理承载基础资源与规则的配置中枢。两页共同构成PC后台的"大脑"与"骨架"。

---

## 页面一：决策中心

### 1. 页面名称与用途

- **页面名称**：决策中心（Decision Center）
- **目标用户**：物业经理（王丽华）、技术主管（赵明远）
- **核心用途**：将分析引擎产出的趋势预测、风险预警、优化建议以可视化方式呈现，支撑管理层从"看数据"到"做决策"的最后一公里。是八步闭环中"分析→计划回流"的关键交互界面。
- **访问路径**：PC后台侧边栏导航 → 决策中心

### 2. 布局结构

采用"卡片式仪表板"布局，信息密度高、层级清晰，适配PC端1280px-1440px宽度。

```
┌─────────────────────────────────────────────────────────────┐
│  顶部导航栏（56px）                                           │
├─────────────────────────────────────────────────────────────┤
│  面包屑：首页 / 决策中心                                       │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐   │
│  │  AI 智能建议卡片区（横向3列，等高）                    │   │
│  │  [卡片1] [卡片2] [卡片3]                               │   │
│  └─────────────────────────────────────────────────────┘   │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────────────────────────┐ ┌─────────────────────┐  │
│  │  风险预警四色图（左，占60%）   │ │ 高频问题TOP10（右，占40%）│  │
│  │  园区地图 + 四色风险区域叠加    │ │ 柱状图 + 环比变化      │  │
│  └──────────────────────────────┘ └─────────────────────┘  │
├─────────────────────────────────────────────────────────────┤
│  ┌─────────────────────────────────────────────────────┐   │
│  │  智能报告生成区（通栏）                                 │   │
│  │  时间范围选择 → 报告类型 → 生成预览 → 导出下载          │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### 3. 关键元素清单

#### 3.1 AI 智能建议卡片区（顶部3列）

| 元素 | 说明 | 数据来源 |
|------|------|---------|
| **卡片1：巡检频次优化建议** | 标题+问题区域名+当前频次+建议频次+预期效果（如"B区3楼消防通道，当前每日1次→建议每日2次，可降低堆物复发率40%"） | 分析引擎：趋势预测+历史复发率 |
| **卡片2：预算调配建议** | 标题+成本类目+当前支出+建议调整+金额变动（如"Q2维保预算：建议向电梯维保倾斜+15%，空调维保缩减-8%"） | 分析引擎：设备故障成本统计+季节性预测 |
| **卡片3：设备更换优先级** | 标题+设备名+故障次数+剩余寿命评估+更换紧迫度（如"3#货梯：近30天故障4次，剩余寿命评估<6个月，建议列入Q3更换计划"） | 分析引擎：设备健康度模型+故障频次聚类 |

- **卡片样式**：白色背景卡片（圆角12px，阴影Shadow-S），顶部左侧彩色竖条标识建议类型（蓝-频次/绿-预算/橙-设备），卡片内文字层级：H3标题+正文描述+辅助文字数据来源
- **操作按钮**：卡片右下角"采纳"（主按钮）+"忽略"（文字按钮）
- **新建议标记**：未读建议卡片左上角红色圆点徽章（10px）

#### 3.2 风险预警四色图（中部左侧）

| 元素 | 说明 |
|------|------|
| **地图容器** | 园区2D底图，占满卡片区域，最小高度400px |
| **四色风险区域** | 红区（P0密度高）/橙区（P1频发）/黄区（P2集中）/蓝区（低风险，正常巡检覆盖） |
| **区域图例** | 地图左下角：四色方块+区域名+问题数量 |
| **区域悬浮卡片** | 鼠标悬停某区域：弹出信息卡片，显示该区域名称、问题总数、TOP3问题类型、环比变化 |
| **时间维度切换** | 地图右上角：近7天/近30天/本季度 三个快捷选项 |

- **四色填充样式**：区域边界线2px实线，内部填充对应功能色20%透明度（红`rgba(239,68,68,0.2)`/橙`rgba(245,158,11,0.2)`/黄`rgba(245,158,11,0.1)`/蓝`rgba(59,130,246,0.1)`）
- **数据来源**：GIS热力图聚类（DBSCAN/OPTICS）+问题点位密度计算

#### 3.3 高频问题区域TOP10（中部右侧）

| 元素 | 说明 |
|------|------|
| **模块标题** | "高频问题区域 TOP10"，H2字号 |
| **柱状图** | 横向柱状图，Y轴为区域名（如"B区-3F-消防通道"），X轴为问题数量，柱体用警告黄填充 |
| **环比变化** | 每根柱体右侧显示环比变化箭头+百分比（↑12%绿色/↓8%红色） |
| **柱体交互** | 点击柱体 → 高亮对应地图区域+下钻该区域问题列表 |
| **问题类型标签** | 柱体下方小字标注该区域TOP1问题类型（如"消防通道堆物"） |

- **图表样式**：柱体圆角4px，柱间距12px，Y轴文字截断显示最多12个字符
- **空状态**：无数据时显示"暂无足够数据，建议积累7天以上巡检记录"

#### 3.4 智能报告生成区（底部通栏）

| 元素 | 说明 |
|------|------|
| **模块标题** | "智能报告生成"，H2字号 |
| **时间范围选择** | 日期选择器（起始-结束），快捷选项：本周/本月/本季度/自定义 |
| **报告类型选择** | 单选：巡检周报 / 巡检月报 / 设备健康报告 / 风险预警报告 |
| **生成按钮** | 主按钮"生成报告"，点击后按钮变为加载态"AI生成中..." |
| **报告预览区** | 生成后以卡片形式展示报告摘要：关键指标行（4个KPI）+核心发现文字+建议列表 |
| **操作按钮组** | "预览完整报告"（次按钮）+ "导出PDF"（主按钮）+ "推送至飞书/钉钉"（图标按钮） |

- **报告卡片样式**：左侧窄条装饰色（科技蓝-500），内部KPI用数据大字（40px）+辅助文字
- **生成耗时预期**：≤10秒，超时提示"报告内容较多，请稍后刷新查看"

### 4. 交互逻辑

#### 4.1 建议采纳/忽略

| 交互 | 行为 | 结果 |
|------|------|------|
| 点击"采纳" | 弹出确认弹窗："采纳后将自动更新巡检计划/预算配置，是否确认？" | 确认后卡片标记"已采纳"（绿色勾选图标），对应配置进入待生效状态，需管理层在"系统管理→规则配置"中最终确认生效时间 |
| 点击"忽略" | 弹出原因选择："已手动处理/暂不适用/信息有误/其他" | 忽略后卡片折叠至"已忽略建议"折叠面板，7天内不再推荐同类建议 |

#### 4.2 区域钻取

| 触发方式 | 行为 | 结果 |
|---------|------|------|
| 点击地图四色区域 | 右侧柱状图高亮对应区域条目，地图放大至该区域（缩放级别+2），底部弹出该区域"问题明细抽屉" | 抽屉内展示该区域近30天所有问题列表（时间/类型/状态/处置人），支持筛选和排序 |
| 点击柱状图条目 | 地图自动定位并高亮对应区域，区域边界闪烁动画（2次，持续1秒） | 同步展示该区域悬浮卡片信息 |

#### 4.3 报告预览/导出

| 步骤 | 行为 |
|------|------|
| 1. 选择时间范围+报告类型 | 表单校验：时间范围≥7天（周报）/≥28天（月报），不满足则提示"时间范围不足，请调整" |
| 2. 点击"生成报告" | 按钮loading态，后台调用AI分析引擎生成报告；生成完成后在预览区展示摘要 |
| 3. 点击"预览完整报告" | 新标签页打开完整报告页面（只读），包含所有图表和数据明细 |
| 4. 点击"导出PDF" | 触发浏览器下载，文件名格式：`巡检周报_2026-04-21_2026-04-27.pdf` |

### 5. AI / GIS 嵌入点标注

| 嵌入点 | 类型 | 具体作用 | 用户感知 |
|--------|------|---------|---------|
| **智能建议卡片内容** | AI | 趋势预测+异常检测+优化建议算法输出 | 卡片以"建议"形式呈现，标注"AI分析基于近30天数据" |
| **风险区域着色** | GIS+AI | DBSCAN空间聚类将问题点位聚合成区域，按P0-P2密度计算风险等级，映射为四色 | 用户在地图上看到直观的风险热力区域 |
| **TOP10区域排序** | AI | 基于问题频次+严重等级加权计算区域风险分值，自动排序 | 柱状图按风险分值降序排列 |
| **报告生成** | AI | NLP自动生成报告摘要文字，趋势预测生成未来7天风险提示 | 报告文字部分标注"由AI辅助生成，请核对后使用" |
| **地图区域悬浮信息** | GIS | 实时查询该区域统计聚合数据（问题数、类型分布、环比） | 鼠标悬停即时展示 |

## 页面二：系统管理

### 1. 页面名称与用途

- **页面名称**：系统管理（System Management）
- **目标用户**：物业经理（王丽华）、技术主管（赵明远）、系统管理员
- **核心用途**：管理园区巡检系统的基础资源——设备台账、人员档案、空间图层、业务规则。是决策建议落地生效的"执行层"，确保数据源头准确、规则配置合理、人员权限清晰。
- **访问路径**：PC后台侧边栏导航 → 系统管理

### 2. 布局结构

采用"侧边标签页+主内容区"的经典后台布局，左侧固定标签导航，右侧动态切换内容面板。

```
┌─────────────────────────────────────────────────────────────┐
│  顶部导航栏（56px）                                           │
├─────────────────────────────────────────────────────────────┤
│  面包屑：首页 / 系统管理                                       │
├──────────┬──────────────────────────────────────────────────┤
│          │                                                  │
│  左侧标签 │  主内容区（随标签切换动态加载）                      │
│  导航栏   │                                                  │
│  (180px) │  ┌──────────────────────────────────────────┐   │
│          │  │  标签内子标签 / 筛选栏 / 操作按钮区          │   │
│  [设备台账]│  ├──────────────────────────────────────────┤   │
│  [人员管理]│  │                                          │   │
│  [空间图层]│  │  内容区（表格/表单/地图）                    │   │
│  [规则配置]│  │                                          │   │
│          │  └──────────────────────────────────────────┘   │
│          │                                                  │
└──────────┴──────────────────────────────────────────────────┘
```

#### 左侧标签导航

| 标签名 | 图标 | 说明 |
|--------|------|------|
| 设备台账 | 工具/设备图标 | 管理园区内所有巡检相关设备 |
| 人员管理 | 人员图标 | 管理巡检员、维修员档案与权限 |
| 空间图层 | 地图/图层图标 | 管理GIS地图底图、POI点位、可视化图层 |

|------|------|

| **配置表格** | 行：超时节点（分派/到场/处置/应急确认）× 列：首次升级（T+30min）/ 二次升级（T+1h）/ 三次升级（T+2h） |
| **单元格内容** | 通知对象多选（巡检员/主管/经理/园区负责人）+ 通知渠道多选（APP推送/短信/电话） |
| **特殊规则区** | 独立卡片："应急事件连续2次不确认→强制升级至L3"（开关控件，默认开启） |

##### 3.4.3 巡检频率规则子标签

| 元素 | 说明 |
|------|------|
| **规则卡片列表** | 每张卡片代表一个区域的一条频率规则：区域名+当前频率+触发条件+调整动作 |
| **示例卡片** | "B区-3楼-消防通道：当前每日1次；触发条件：近7天同类问题≥3次；自动调整：提升至每日2次" |
| **操作按钮** | 卡片内"编辑"/"停用"/"删除"；页面右上角"新增规则"（主按钮） |
| **规则状态** | 生效中（成功绿标签）/ 已停用（文字禁用色标签） |
| **决策回流提示** | 来自"决策中心"已采纳的建议，在此以"待生效建议"列表形式展示，点击"立即生效"则转为正式规则 |

### 4. 交互逻辑

#### 4.1 标签切换

| 交互 | 行为 | 结果 |
|------|------|------|
| 点击左侧标签导航 | 主内容区切换为对应标签内容，标签激活态变化 | URL同步更新哈希（如`#/system/equipment`），支持浏览器前进后退 |
| 标签内子标签切换 | 使用PC端标签页组件（Tab），位于主内容区顶部 | 子标签切换保留当前筛选状态（不重置搜索条件） |

#### 4.2 表单编辑

| 交互 | 行为 | 结果 |
|------|------|------|
| 表格行内编辑 | 点击"编辑" → 行变为表单 → 修改 → 点击"保存" | 校验通过后更新数据，行恢复展示态，顶部toast提示"保存成功"；校验失败则在字段下方显示红色错误文字 |
| 弹窗表单（新增） | 点击"新增" → 弹出模态弹窗（宽度640px）→ 填写表单 → 提交 | 弹窗内支持分步表单（设备新增：基础信息→位置→参数→确认），最后一步展示信息摘要供核对 |
| 删除确认 | 点击"删除" → 二次确认弹窗："删除后关联记录将保留但不可追溯，是否确认？" | 确认后删除，取消后无变化 |

#### 4.3 地图联动

| 交互 | 行为 | 结果 |
|------|------|------|
| 空间图层 → POI表格联动 | 点击地图POI标记 | 右侧/下方POI表格自动滚动定位至对应行，该行背景高亮（科技蓝-100），持续3秒后恢复 |
| POI表格 → 地图联动 | 鼠标悬停表格行 | 地图对应POI标记放大1.5倍+脉冲动画（科技蓝-500圆环扩散） |
| 设备台账 → 空间图层跳转 | 设备详情页点击"在地图中查看" | 跳转至空间图层标签，自动开启POI点位层，地图中心定位至该设备，标记点高亮 |
| 图层开关 | 切换任一图层开关 | 地图即时响应，开关过渡动画300ms；若数据量大（如轨迹回放），按钮loading态并提示"数据加载中" |

### 5. AI / GIS 嵌入点标注

| 嵌入点 | 类型 | 具体作用 | 用户感知 |
|--------|------|---------|---------|
