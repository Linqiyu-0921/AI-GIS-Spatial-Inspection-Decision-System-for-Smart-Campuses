# CLAUDE.md — proj-smart-campus-inspection

## 项目概述

智慧园区空间巡检决策系统 — 产品方案文档项目（非代码项目）。

本项目仅产出产品方案文档集（PRD、架构、竞品分析、用户研究、设计稿），不包含任何可运行代码。所有技术栈标注为 `[假设]`。

## 核心产品约束

- **园区类型**：办公商务园区（写字楼集群、CBD）
- **用户角色**：2 个 — 巡检员（移动端）+ 管理层（PC 后台）
- **巡检场景**：3 个 — 日常巡检/设备检查/应急事件
- **AI 能力**：3 个 — 图像识别/NLP 结构化/预测分析
- **GIS 能力**：3 个 — 空间可视化/问题聚类/缓冲区分析
- **工单流程**：4 步 — 创建→分配→处置→验收
- **地图**：V1.0 仅 2D，V2.0 才做 3D/数字孪生

## 命名规范

- 所有文件和目录使用 `kebab-case`
- 文档文件以数字前缀排序：`01-xxx.md`, `02-xxx.md`

## 写作规范

- 流程图使用文字描述 + 表格，不依赖外部绘图工具
- 所有假设标注 `[假设]` 前缀
- SLA 时限使用具体数字（如"≤15min"），不用模糊词
- 引用竞品分析时注明来源

## 禁止事项

- 不写任何代码文件（.js/.ts/.vue/.py 等）
- 不创建 package.json 或任何依赖配置
- 不搭建服务器或数据库
- 不扩展超过 3 个场景 / 2 个角色 / 3 个 AI 能力 / 3 个 GIS 能力
