# 智慧园区空间巡检决策系统

> 基于 AI 大模型与 GIS 的办公商务园区巡检决策系统产品方案。

本项目聚焦办公商务园区（写字楼集群、CBD）的巡检管理痛点，以 GIS 空间决策为核心引擎、以 AI 全流程嵌入为技术底座、以自优化闭环为进化机制，构建从"记录巡了什么"到"决策该怎么巡"的巡检决策系统。

## 项目概述

### 背景与痛点

中国智慧园区市场规模预计 2026 年突破 3000 亿元，其中运维管理（含巡检）占整体投入的 20%-25%。然而，当前办公商务园区巡检管理普遍存在以下结构性问题：

| 痛点                   | 现状                                         | 后果                                   |
| ---------------------- | -------------------------------------------- | -------------------------------------- |
| **巡检靠腿、记录靠纸** | 巡检员手持纸质工单逐点打卡，发现问题手写记录 | 信息滞后、描述模糊、无法追溯           |
| **发现问题≠解决问题**  | 巡检记录与工单系统割裂，需人工转录、手动派单 | 问题遗漏、响应延迟、责任不清           |
| **巡了什么≠该巡什么**  | 巡检路线和频率固定，不根据历史数据动态调整   | 高风险区域巡检不足，低风险区域重复浪费 |
| **数据沉睡**           | 巡检数据仅用于存档，不做趋势分析和决策支持   | 管理层无法预判风险，决策依赖经验       |

### 产品愿景

> 让每一次巡检都有据可依、每一个问题都闭环解决、每一条数据都驱动决策。

### 核心目标（V1.0）

| 编号 | 目标                         | 量化指标                        |
| ---- | ---------------------------- | ------------------------------- |
| G1   | 问题发现到工单创建的时间缩短 | 从人工转录的 30min 缩短至 ≤2min |
| G2   | 工单闭环率提升               | 从 70% 提升至 ≥95%              |
| G3   | 巡检覆盖率提升               | 从 60% 提升至 ≥90%              |
| G4   | 管理决策有数据支撑           | 100% 巡检数据进入分析引擎       |
| G5   | 系统易用性达标               | 巡检员上手培训时间 ≤30min       |

### 目标用户

| 角色       | 使用端         | 核心诉求                                   |
| ---------- | -------------- | ------------------------------------------ |
| **巡检员** | 移动端（APP）  | 快速接收任务、高效上报问题、减少重复劳动   |
| **管理层** | PC 后台（Web） | 掌控全局态势、追踪问题闭环、基于数据做决策 |

## 核心功能

### 三大巡检场景

| 场景         | 定义                                                           | 频率        | 触发方式                |
| ------------ | -------------------------------------------------------------- | ----------- | ----------------------- |
| **日常巡检** | 按计划路线对园区公共区域进行例行巡查，发现并上报环境、安全隐患 | 每日 1-2 次 | 计划任务自动下发        |
| **设备检查** | 对电梯、空调、消防、照明等关键设备进行专项检查，记录运行状态   | 每周/每月   | 计划任务 + 设备告警触发 |
| **应急事件** | 火灾、水管爆裂、电梯困人等突发事件的快速响应与处置             | 不定期      | 人工上报 / 系统检测     |

### 七步闭环流程

所有场景共享统一的闭环流程骨架：

```
计划 → 执行 → 发现 → 工单 → 处置 → 验收 → 分析
```

| 步骤       | 名称               | AI 嵌入                              | GIS 嵌入                      |
| ---------- | ------------------ | ------------------------------------ | ----------------------------- |
| **① 计划** | 巡检计划编排与下发 | 基于历史数据推荐巡检频率和路线       | 覆盖分析：检测巡检盲区        |
| **② 执行** | 现场巡检执行       | 异常行为预警                         | 轨迹记录：GPS 定位 + 路线回放 |
| **③ 发现** | 问题发现与上报     | 图像识别自动分类；NLP 结构化上报内容 | 问题点位自动标注到地图        |
| **④ 工单** | 工单创建与分派     | 智能推荐处置人                       | 就近匹配：计算最近可用人员    |
| **⑤ 处置** | 问题处置与反馈     | 处置方案知识库推荐                   | 导航指引至问题点位            |
| **⑥ 验收** | 处置结果验收       | 图像比对验证修复效果                 | 定位验证是否在问题点位完成    |
| **⑦ 分析** | 数据分析与决策优化 | 趋势预测、异常检测                   | 热力图聚类：空间维度问题分布  |

### 核心能力矩阵

| 能力层       | 功能说明                                                                                                                      |
| ------------ | ----------------------------------------------------------------------------------------------------------------------------- |
| **AI 引擎**  | 图像识别（缺陷分类）、NLP 结构化（语音/文字）、预测分析（趋势/推荐）、图像比对（验收验证）、智能推荐（处置人/方案）、异常检测 |
| **GIS 引擎** | 空间可视化（2D 地图）、轨迹记录与回放、聚类分析（热力图）、覆盖分析（盲区检测）、就近匹配（人员/资源）、导航指引（路径规划）  |
| **SLA 管控** | 工单分级（P0-P3）、时限配置、超时自动升级与转派、SLA 达成率看板                                                               |

## 竞品差异化

### 核心竞争优势

1. **GIS 空间决策引擎**：从"地图展示"升级到"空间驱动"，通过问题热力图聚类、覆盖盲区检测、缓冲区风险建模、空间趋势预测，实现"在哪里"到"为什么在这里"的跨越。

2. **原生七步闭环 + SLA 驱动**：巡检发现问题自动触发工单，工单完成自动触发复核，全流程内置 SLA 时间窗口和超时自动升级策略，确保问题不仅被记录，而且被按时解决。

3. **AI 全流程嵌入 + 自优化回流**：AI 嵌入巡检全流程每个环节（识别、结构化、分派、分析、决策），并通过自优化回流闭环，让系统基于历史数据自动调整巡检频率、路线和策略。

### 竞争格局

| 竞品         | 优势                           | 劣势                                         |
| ------------ | ------------------------------ | -------------------------------------------- |
| **海康威视** | 安防硬件基座，视频 AI 能力领先 | 软件碎片化、GIS 薄弱、流程闭环依赖人工       |
| **广联达**   | BIM 核心壁垒，施工数字化强     | 场景绑定过深、GIS 空间分析缺失、SLA 管控空白 |
| **开源方案** | 扩展性强、无供应商锁定         | 无 AI、无空间分析、无自优化                  |

> **竞争空白**：AI 深度嵌入巡检全流程、GIS 空间决策驱动巡检策略、自优化闭环持续进化——这三点在商业产品和开源领域均未被解决。

## 贡献规范

### 文档写作规范

- 每个文档开头包含 `# 标题` 和一段简要说明
- 流程图使用文字描述 + 表格，不依赖外部绘图工具
- 所有假设标注 `[假设]` 前缀
- SLA 时限使用具体数字（如"≤15min"），不用模糊词
- 引用竞品分析时注明来源

### 命名规范

- 所有文件和目录使用 `kebab-case`
- 文档文件以数字前缀排序：`01-xxx.md`, `02-xxx.md`
- 图片文件以场景前缀命名：`daily-inspection-flow.png`

### 禁止事项

- 不写任何代码文件（.js/.ts/.vue/.py 等）
- 不创建 package.json 或任何依赖配置
- 不搭建服务器或数据库
- 不扩展超过 3 个场景 / 2 个角色 / 3 个 AI 能力 / 3 个 GIS 能力
