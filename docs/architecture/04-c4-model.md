### 1.2 人与系统的关系

| 角色 | 职责 | 与系统的交互 | 使用端 |
|------|------|-------------|--------|
| **巡检员** | 执行现场巡检任务，发现问题并上报 | 接收巡检计划 → 执行巡检 → 上报问题 → 接收处置工单 → 反馈处置结果 | 移动端 |
| **管理层** | 监控园区运营状态，调度工单，分析决策 | 查看数据大盘 → 调度工单 → 确认优化建议 → 管理巡检配置 | PC端 |
| **维修员** | 接收并处置工单，修复现场问题 | 接收工单通知 → 导航到场 → 执行维修 → 拍照留证 → 提交完成 | 移动端 |
| **指挥中心（V2.0）** | 应急事件统一指挥调度 | 接收应急告警 → 资源协调 → 态势监控 → 指令下发 | PC大屏端 |

### 1.3 外部系统与系统的关系

| 外部系统 | 交互方向 | 交互内容 | 版本 |
|---------|---------|---------|------|
| **IoT传感器（V2.0）** | → 系统 | 电梯运行状态、消防烟感告警、空调温度异常等实时数据流 | V2.0 |
| **地图服务（高德API）** | ↔ 系统 | 2D矢量底图瓦片、POI数据、步行/驾车路径规划、地理编码 | V1.0 |
| **AI云服务（阿里云）** | → 系统 | 图像识别结果、语音转文字、NLP信息抽取、预测模型 | V1.0 |
| **BIM模型（V2.0）** | → 系统 | 3D建筑模型数据、设备空间坐标、楼层结构信息 | V2.0 |
| **企业IM（V2.0）** | ← 系统 | 工单状态通知、应急告警推送、日报/周报推送 | V2.0 |
| **第三方工单系统（V2.0）** | ↔ 系统 | 工单数据同步、状态双向回写 | V2.0 |

### 1.4 外部系统依赖说明

| 外部系统 | 依赖强度 | 可替代性 | 说明 |
|---------|---------|---------|------|
| **高德地图API** | 强依赖 | 中 | V1.0核心功能（导航、地图渲染）依赖，可替换为百度/天地图 |
| **阿里云视觉智能** | 强依赖 | 中 | V1.0图像识别核心能力，可替换为百度/腾讯云AI |
| **阿里云NLP** | 强依赖 | 高 | 语音转写和NLP抽取，可替换为讯飞、百度NLP或自研 |
| **IoT传感器** | 弱依赖 | — | V2.0功能，V1.0不影响核心流程 |
| **BIM模型** | 弱依赖 | — | V2.0功能，V1.0仅使用2D矢量地图 |
| **企业IM** | 弱依赖 | 高 | V2.0通知渠道，系统内已有推送能力 |

### 2.2 容器职责说明

#### 2.2.1 移动App容器

| 属性 | 说明 |
|------|------|
| **技术栈 [假设]** | Flutter / React Native |
| **部署形态** | iOS/Android 原生应用包 |
| **核心职责** | 为巡检员和维修员提供移动端现场作业能力 |
| **关键能力** | 离线任务缓存、GPS轨迹采集、相机调用、语音录入、推送通知接收 |
| **对外交互** | ←→ API网关（HTTPS/WebSocket） |
| **内部构成** | 任务模块、巡检模块、AI交互模块、LBS模块、工单模块、消息模块（不展开到Component） |

#### 2.2.2 Web后台容器

| 属性 | 说明 |
|------|------|
| **技术栈 [假设]** | Vue 3 / React + 图表库(ECharts/AntV) + GIS库(Leaflet/Mapbox GL) |
| **部署形态** | Web SPA，部署于Nginx静态服务器 |
| **核心职责** | 为管理层提供数据可视化、工单调度、决策分析和系统管理能力 |
| **关键能力** | 数据大盘渲染、GIS地图交互、工单流转管理、报告生成与导出 |
| **对外交互** | ←→ API网关（HTTPS/WebSocket） |
| **内部构成** | 数据大盘模块、工单调度台模块、决策中心模块、GIS后台模块、系统管理模块（不展开到Component） |

#### 2.2.3 API网关容器

| 属性 | 说明 |
|------|------|
| **技术栈 [假设]** | Spring Cloud Gateway / Nginx + Lua |
| **部署形态** | 与业务应用同进程（V1.0），2节点负载均衡 |
| **核心职责** | 统一入口、认证鉴权、路由分发、流量控制 |
| **关键能力** | JWT校验、请求路由、限流熔断、日志记录、跨域处理 |
| **对外交互** | ← 移动App / Web后台；→ 业务应用容器 / AI服务容器 / GIS服务容器 |

#### 2.2.4 业务应用容器

| 属性 | 说明 |
|------|------|
| **技术栈 [假设]** | Java（Spring Boot）/ Node.js（NestJS） |
| **部署形态** | 单体应用，2节点负载均衡 |
| **核心职责** | 承载七步闭环业务流程、用户管理、通知推送、定时任务 |
| **关键能力** | 计划编排、工单状态机、SLA计算、推送通知、数据报表生成 |
| **对外交互** | ←→ 业务数据库容器；←→ 缓存容器；→ 文件存储容器 |
| **内部构成** | 计划服务、执行服务、工单服务、验收服务、分析服务、用户服务、通知服务（不展开到Component） |

#### 2.2.5 AI服务容器

| 属性 | 说明 |
|------|------|
| **技术栈 [假设]** | Python（FastAPI/Flask）+ OpenCV / TensorFlow / PyTorch |
| **部署形态** | 独立进程，可与业务应用同机部署 |
| **核心职责** | 提供图像识别、NLP结构化、预测分析、图像比对、异常检测等AI能力 |
| **关键能力** | 图像分类与缺陷识别、语音转文字、实体抽取、趋势预测、图像相似度比对 |
| **对外交互** | ←→ 业务应用容器；→ 阿里云视觉智能 / 阿里云NLP（第三方API） |
| **内部构成** | 图像识别服务、NLP服务、预测分析服务、图像比对服务、异常检测服务（不展开到Component） |

#### 2.2.6 GIS服务容器

| 属性 | 说明 |
|------|------|
| **技术栈 [假设]** | Java（Spring Boot）/ Go + PostGIS + GeoTools |
| **部署形态** | 独立进程，可与业务应用同机部署 |
| **核心职责** | 提供地图渲染代理、空间分析、路径规划、轨迹处理等GIS能力 |
| **关键能力** | 瓦片代理与缓存、空间查询与计算、路径规划代理、轨迹聚合、DBSCAN聚类 |
| **对外交互** | ←→ 业务应用容器；←→ 空间数据库容器；→ 高德地图API（第三方API） |
| **内部构成** | 地图代理服务、空间分析服务、路径规划服务、轨迹服务、聚类服务（不展开到Component） |

### 2.3 数据容器职责说明

| 容器 | 技术栈 [假设] | 核心职责 | 关键数据 |
|------|--------------|---------|---------|
| **业务数据库** | PostgreSQL 14+（主从复制） | 存储业务实体关系数据 | 用户、巡检计划、工单、验收、报告、配置 |
| **空间数据库** | PostgreSQL + PostGIS 3.1+ | 存储空间几何数据，支持空间查询 | 轨迹点、问题点位、路线、聚类结果、覆盖区域 |
| **文件存储** | 阿里云OSS / MinIO | 存储非结构化大文件 | 照片、语音、视频、报告PDF |
| **缓存** | Redis 6+（哨兵模式） | 加速热点数据访问，支持实时场景 | 用户会话、实时位置、热点报表、地图瓦片 |

## 三、容器间交互关系

### 3.1 同步调用关系

| 调用方 | 被调用方 | 协议 | 用途 |
|--------|---------|------|------|
| 移动App | API网关 | HTTPS/WebSocket | 业务请求、实时位置推送 |
| Web后台 | API网关 | HTTPS/WebSocket | 管理操作、数据查询、地图交互 |
| API网关 | 业务应用 | HTTP | 路由转发后的业务处理 |
| 业务应用 | AI服务 | HTTP | 图像识别、NLP、预测分析请求 |
| 业务应用 | GIS服务 | HTTP | 空间查询、路径规划、轨迹存储 |
| AI服务 | 阿里云AI | HTTPS | 第三方AI能力调用 |
| GIS服务 | 高德API | HTTPS | 地图瓦片、路径规划调用 |

### 3.2 数据存储关系

| 写入方 | 存储容器 | 数据内容 |
|--------|---------|---------|
| 业务应用 | 业务数据库 | 所有业务实体数据 |
| GIS服务 | 空间数据库 | 轨迹、点位、路线、聚类结果 |
| 业务应用 / 移动App | 文件存储 | 照片、语音、视频、报告文件 |
| 业务应用 / GIS服务 | 缓存 | 会话、位置、热点数据、瓦片 |

## 四、C4 模型设计约束

| 约束项 | 说明 |
|--------|------|
