# 01 系统架构设计

> 基于七步闭环流程和AI+GIS双引擎能力，定义智慧园区空间巡检决策系统的分层架构、模块划分及版本边界。

---

## 一、分层架构总览

系统采用四层架构：展示层 → 业务层 → 能力层 → 数据层。

```
┌─────────────────────────────────────────────────────────────────────────┐
│                              展示层 (Presentation)                       │
│  ┌─────────────────────────────┐    ┌─────────────────────────────────┐ │
│  │      移动端 — 巡检员端        │    │        PC端 — 管理层端           │ │
│  │  · 任务中心                  │    │  · 数据大盘                      │ │
│  │  · 场景化巡检（日常/设备/应急）│    │  · 工单调度（闭环）              │ │
│  │  · AI辅助识别                │    │  · 决策中心                      │ │
│  │  · LBS服务（定位/导航）       │    │  · 系统管理                      │ │
│  └─────────────────────────────┘    └─────────────────────────────────┘ │
├─────────────────────────────────────────────────────────────────────────┤
│                              业务层 (Business)                           │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌────────┐│
│  │ ① 计划   │→│ ② 执行   │→│ ③ 发现   │→│ ④ 工单   │→│ ⑤ 处置   │→│ ⑥ 验收  ││
│  │ 编排下发  │ │ 现场巡检  │ │ 上报识别  │ │ 创建分派  │ │ 处置反馈  │ │ 结果确认 ││
│  └─────────┘ └─────────┘ └─────────┘ └─────────┘ └─────────┘ └────────┘│
│                              ↓                                          │
│                         ┌─────────┐                                     │
│                         │ ⑦ 分析   │                                     │
│                         │ 数据决策  │                                     │
│                         └────┬────┘                                     │
│                              │ 数据回流自优化                              │
└──────────────────────────────┼──────────────────────────────────────────┘
                               ↓
┌─────────────────────────────────────────────────────────────────────────┐
│                              能力层 (Capability)                         │
│  ┌─────────────────────────────────┐  ┌─────────────────────────────────┐│

│  │  · 预测分析（趋势/推荐）          │  │  · 聚类分析（热力图）             ││
│  │  · 图像比对（验收验证）           │  │  · 覆盖分析（盲区检测）           ││
│  │  · 智能推荐（处置人/方案）        │  │  · 就近匹配（人员/资源）          ││
│  │  · 异常检测                      │  │  · 导航指引（路径规划）           ││
│  └─────────────────────────────────┘  └─────────────────────────────────┘│
├─────────────────────────────────────────────────────────────────────────┤
│                              数据层 (Data)                               │
│  ┌──────────────┐ ┌──────────────┐ ┌──────────────┐ ┌──────────────────┐│
│  │  矢量地图数据   │ │  BIM数据      │ │  IoT数据      │ │   业务数据        ││
│  │  (园区2D底图)   │ │  (V2.0)       │ │  (V2.0)       │ │   (巡检/工单/用户) ││
```

## 二、模块划分

### 2.1 移动端 — 巡检员端

| 模块 | 功能 | V1.0 | V2.0 |
|------|------|------|------|
| **任务中心** | 接收当日巡检计划、查看任务列表、历史任务查询 | ✅ | — |
| **场景化巡检** | 日常巡检导航+checklist、设备检查逐项录入、应急事件上报 | ✅ | — |
| **AI辅助** | 拍照识别问题类型、语音转文字、NLP结构化、处置方案推荐 | ✅ | 扩展IoT异常识别 |
| **LBS服务** | GPS定位、轨迹记录、GIS导航、问题点位打标 | ✅ | 室内定位增强 |
| **工单处置** | 接收处置工单、到场打卡、处置反馈、拍照留证 | ✅ | — |
| **即时通讯** | 工单状态推送、管理层通知、应急广播 | ✅ | 音视频通话 |

### 2.2 PC端 — 管理层端

| 模块 | 功能 | V1.0 | V2.0 |
|------|------|------|------|
| **数据大盘** | 巡检完成率、工单闭环率、SLA达成率、实时位置一屏概览 | ✅ | 大屏适配 |
| **工单调度** | 工单列表、分派确认、超时督办、升级规则配置 | ✅ | 自动分派 |
| **决策中心** | 问题热力图、趋势预测报告、优化建议确认、数据下钻 | ✅ | 多园区对比 |
| **系统管理** | 用户权限、巡检点位配置、SLA模板、应急预案管理 | ✅ | 开放API管理 |
| **GIS后台** | 路线规划、覆盖分析、缓冲区设置、轨迹回放 | ✅ | 3D可视化 |

### 2.3 共享层 — 双引擎与数据底座

| 模块 | 功能 | V1.0 | V2.0 |
|------|------|------|------|
| **AI引擎** | 图像识别、NLP、预测分析、推荐算法、异常检测 | ✅ | 模型自训练 |
| **GIS引擎** | 空间计算、地图渲染、路径规划、聚类分析、覆盖分析 | ✅ | 3D/BIM渲染 |
| **数据层** | 矢量地图存储、业务数据存储、文件存储（照片/语音） | ✅ | 时序数据库 |

## 三、AI 嵌入点分布

| 闭环环节 | 嵌入模块 | AI能力 | 输入 | 输出 |
|---------|---------|--------|------|------|
| **① 计划** | AI引擎-预测分析 | 历史数据挖掘，推荐巡检频次和路线 | 30天历史巡检记录、问题分布 | 优化后的巡检计划建议 |
| **② 执行** | AI引擎-异常检测 | 实时行为预警（跳过checklist/偏离路线） | 操作行为序列、GPS轨迹 | 预警提示 |
| **③ 发现** | AI引擎-图像识别+NLP | 拍照识别问题类型、语音结构化 | 照片、语音/文字描述 | 问题类型、结构化工单字段 |
| **④ 工单** | AI引擎-智能推荐 | 推荐最优处置人（技能+位置+负载） | 问题类型、处置人档案、实时位置 | Top3处置人候选列表 |
| **⑤ 处置** | AI引擎-知识库推荐 | 匹配历史相似工单的处置方案 | 问题类型、历史工单库 | 推荐处置方案 |
| **⑥ 验收** | AI引擎-图像比对 | 处置前后照片对比，验证修复效果 | 上报照片、处置后照片 | 修复判定结果+置信度 |
| **⑦ 分析** | AI引擎-趋势预测+异常检测 | 预测风险区域、识别异常模式 | 全流程历史数据 | 预测报告、异常告警、优化建议 |

## 四、GIS 嵌入点分布

| 闭环环节 | 嵌入模块 | GIS能力 | 输入 | 输出 |
|---------|---------|--------|------|------|
| **① 计划** | GIS引擎-覆盖分析 | 检测巡检盲区，验证路线全覆盖 | 园区边界、巡检路线、点位分布 | 盲区告警、覆盖报告 |
| **② 执行** | GIS引擎-轨迹记录 | GPS采样+路线回放 | 实时GPS坐标（10s间隔） | 巡检轨迹、路线回放数据 |
| **③ 发现** | GIS引擎-定位打标 | 问题点位自动标注到地图 | GPS坐标、点位关联规则 | 地图标记图层、空间关联信息 |
| **④ 工单** | GIS引擎-就近匹配 | 计算最近可用处置人员 | 问题点位坐标、处置人实时坐标 | 距离排序列表 |
| **⑤ 处置** | GIS引擎-导航指引 | 路径规划至问题点位 | 起点坐标、终点坐标 | 导航路径（步行/驾车） |
| **⑥ 验收** | GIS引擎-现场确认 | 验证处置地点与上报地点一致性 | 验收时GPS坐标、问题点位坐标 | 距离验证结果 |
| **⑦ 分析** | GIS引擎-热力图聚类 | DBSCAN/OPTICS空间聚类 | 历史问题点位集合 | 热力图图层、聚类中心 |

## 六、V1.0 / V2.0 版本边界

### 6.1 V1.0 范围（MVP）

| 维度 | V1.0 内容 |
|------|----------|
| **地图** | 2D矢量地图，室外GPS定位 |
| **AI** | 图像识别、NLP结构化、基础预测分析 |
| **GIS** | 轨迹记录、定位打标、热力图聚类、覆盖分析、就近匹配、导航 |
| **数据** | 矢量地图 + 业务数据库 + 文件存储 |
| **场景** | 日常巡检、设备检查、人工上报应急 |
| **角色** | 巡检员（移动端）+ 管理层（PC端） |

### 6.2 V2.0 扩展（规划）

| 维度 | V2.0 扩展 |
|------|----------|
| **地图** | BIM/3D数字孪生、室内定位增强 |
| **AI** | IoT数据融合分析、模型自训练优化、视频流实时识别 |
| **GIS** | 3D空间可视化、缓冲区动态分析、多园区空间对比 |
| **数据** | IoT时序数据库接入、BIM模型存储、外部系统数据 |
| **场景** | IoT自动告警触发应急、无人机巡检联动 |
| **集成** | 开放API、第三方系统对接（门禁/电梯/消防） |

## 七、技术约束声明

| 约束项 | 说明 |
|--------|------|
| **不做微服务拆分** | V1.0采用单体应用架构，降低部署复杂度；微服务化为V2.0技术演进选项 |
| **不做到类/接口级别** | 本架构设计聚焦模块边界和数据交互，不展开类图、接口定义、序列图 |
| **不写代码** | 架构文档仅定义模块职责和交互契约，不包含实现代码 |
| **2D优先** | V1.0地图渲染仅支持2D矢量底图，3D/BIM为V2.0扩展 |
| **单园区** | V1.0数据模型按单园区设计，多园区租户隔离为V2.0扩展 |
