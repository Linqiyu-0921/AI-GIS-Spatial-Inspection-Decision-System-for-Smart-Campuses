# 03 场景概述

> 定义系统覆盖的3个核心巡检场景，并建立统一的七步闭环流程骨架，为后续详细场景设计提供框架。

---

## 一、场景一句话定义

| 场景 | 定义 | 频率 | 触发方式 |
|------|------|------|---------|
| **日常巡检** | 巡检员按计划路线对园区公共区域进行例行巡查，发现并上报环境、安全隐患 | 每日1-2次 | 计划任务自动下发 |
| **设备检查** | 对电梯、空调、消防、照明等关键设备进行专项检查，记录运行状态 | 每周/每月（按设备类型） | 计划任务 + 设备告警触发 |
| **应急事件** | 火灾、水管爆裂、电梯困人等突发事件的快速响应与处置 | 不定期 | 人工上报 / IoT告警 / 系统检测 |

---

## 二、七步闭环流程骨架

所有巡检场景共享统一的七步闭环流程，确保"发现→解决→反馈"的完整性：

```
计划 → 执行 → 发现 → 工单 → 处置 → 验收 → 分析
```

### 步骤详解

| 步骤 | 名称 | 角色 | AI嵌入 | GIS嵌入 |
|------|------|------|--------|---------|
| **① 计划** | 巡检计划编排与下发 | 管理层编排，巡检员接收 | 基于历史数据推荐巡检频率和路线（自优化） | 覆盖分析：检测巡检盲区，确保无遗漏 |
| **② 执行** | 现场巡检执行 | 巡检员按路线逐点巡检 | 巡检进度实时同步，异常区域预警 | 轨迹记录：GPS定位+路线回放，确保真实到达 |
| **③ 发现** | 问题发现与上报 | 巡检员拍照/语音/文字上报 | 图像识别自动分类缺陷类型；NLP结构化上报内容 | 定位打标：问题点位自动标注到GIS地图 |
| **④ 工单** | 工单创建与分派 | 系统自动创建，管理层分派 | 智能推荐处置人（基于技能+位置+负载） | 就近匹配：GIS计算最近可用处置人员 |
| **⑤ 处置** | 问题处置与反馈 | 处置人（巡检员/外包/维修队） | 处置方案知识库推荐 | 导航指引：GIS导航至问题点位 |
| **⑥ 验收** | 处置结果验收 | 管理层或巡检员验收 | 图像比对：处置前后照片AI对比验证 | 现场确认：定位验证是否在问题点位完成 |
| **⑦ 分析** | 数据分析与决策优化 | 管理层查看，系统自动产出 | 趋势预测、异常检测、优化建议 | 热力图聚类：空间维度的问题分布分析 |

### 闭环分支逻辑

并非所有问题都需要走完整七步。根据严重程度，存在两条分支：

| 分支 | 适用场景 | 流程 | 说明 |
|------|---------|------|------|
| **轻微当场闭环** | 地面有纸屑、灯泡需更换等 | 发现 → 当场处置 → 验收 → 分析 | 巡检员自行处理，拍照留证即可 |
| **严重走工单** | 设备故障、安全隐患、结构损坏等 | 发现 → 工单 → 分派 → 处置 → 验收 → 分析 | 需要专业人员处理，走完整工单流程 |

判断标准由AI辅助：图像识别问题类型 → 匹配严重等级 → 引导巡检员选择分支。

## 三、场景与闭环的映射

### 3.1 日常巡检

| 闭环步骤 | 日常巡检中的具体表现 |
|---------|-------------------|
| 计划 | 管理层设定巡检路线、频率、点位；系统根据历史数据建议优化（如某区域连续30天无问题，降频至每周2次） |
| 执行 | 巡检员按"导航式"路线逐点巡检，每到一个点位完成checklist |
| 发现 | 发现消防通道堆物 → 拍照 → AI识别为"消防隐患-中等" → 语音补充描述 → NLP结构化为工单字段 |
| 工单 | 系统自动创建P1工单 → AI推荐最近可用保洁人员 → 管理层确认分派 |
| 处置 | 保洁人员收到工单 → 导航至消防通道 → 清理杂物 → 拍照上传处置结果 |
| 验收 | 巡检员下次巡检时复核 → 或管理层远程验收处置照片 |
| 分析 | 月度分析：消防通道堆物问题集中在B区3楼 → 建议增加该区域巡查频次或增设监控 |

### 3.2 设备检查

| 闭环步骤 | 设备检查中的具体表现 |
|---------|-------------------|
| 计划 | 按设备维保规范设定检查周期（电梯月检、消防季检、空调半年检）；系统自动排期 |
| 执行 | 巡检员/技术员到达设备点位 → 按设备checklist逐项检查（运行参数、外观、异常声音等） |
| 发现 | 发现C区2号电梯运行异响 → 录音+拍照 → AI识别为"设备异常-高" → 自动提升严重等级 |
| 工单 | 系统创建P0工单 → 自动通知技术主管 → AI推荐电梯维保单位 → SLA设为4小时 |
| 处置 | 维保单位接单 → 现场检修 → 更换零部件 → 上传维修报告 |
| 验收 | 技术主管审核维修报告 → 安排复检 → 确认电梯恢复正常运行 |
| 分析 | 趋势分析：C区电梯近3个月故障率上升40% → 建议安排大修或更换 → 进入预算审批流程 |

### 3.3 应急事件

| 闭环步骤 | 应急事件中的具体表现 |
|---------|-------------------|
| 计划 | 预设应急预案（火灾、水管爆裂、电梯困人等）；系统自动匹配响应流程 |
| 执行 | 事件触发 → 系统自动识别事件类型 → 推送应急预案 → 通知相关人员 |
| 发现 | 巡检员发现水管爆裂 → 紧急上报 → AI识别为"应急事件-水管爆裂" → 自动触发应急流程 |
| 工单 | 系统创建P0应急工单 → 同时通知：物业经理+维修队+安保 → GIS定位最近可用人员 |
| 处置 | 维修队关闭对应水管阀门 → 安保疏散周边人员 → 物业经理协调资源 → 处置过程实时更新 |
| 验收 | 水管修复 → 积水清理 → 现场恢复 → 管理层确认事件关闭 |
| 分析 | 事件复盘：从发现到阀门关闭耗时 [假设] 25min → 分析瓶颈 → 优化响应流程 → 更新应急预案 |

## 四、应急响应分级机制

应急事件触发后，按严重程度分级响应：

| 级别 | 定义 | 响应时限 | 示例 | 升级规则 |
|------|------|---------|------|---------|
| **P0** | 重大安全事件，需立即响应 | ≤15min | 火灾、电梯困人、水管爆裂 | 15min未响应自动升级至园区负责人 |
| **P1** | 严重隐患，需尽快处理 | ≤2h | 消防通道堵塞、设备重大故障 | 2h未处理自动升级至技术主管 |
| **P2** | 一般问题，按计划处理 | ≤24h | 照明损坏、墙面污损 | 24h未处理自动升级至物业经理 |
| **P3** | 轻微问题，可批量处理 | ≤72h | 标识牌脱落、绿化缺失 | 72h未处理提醒物业经理 |

> 引用来源：竞品分析指出海康和广联达均缺乏SLA驱动机制，本项目以SLA为核心差异点。

## 五、AI与GIS在场景中的嵌入点总览

### AI嵌入点

| 环节 | AI能力 | 具体作用 |
|------|--------|---------|
| 计划 | 预测分析 | 基于历史数据推荐巡检频率和路线优化 |
| 执行 | 实时预警 | 异常区域实时提醒（如连续多日未巡检的点位） |
| 发现 | 图像识别 + NLP | 自动识别缺陷类型 + 结构化上报内容 |
| 工单 | 智能推荐 | 基于技能、位置、负载推荐最佳处置人 |
| 处置 | 知识库推荐 | 匹配历史类似问题的处置方案 |
| 验收 | 图像比对 | 处置前后照片AI对比，自动判断是否修复 |
| 分析 | 趋势预测 + 异常检测 | 预测风险区域、识别异常模式 |

### GIS嵌入点

| 环节 | GIS能力 | 具体作用 |
|------|---------|---------|
| 计划 | 覆盖分析 | 检测巡检盲区，确保路线全覆盖 |
| 执行 | 轨迹记录 | GPS定位+路线回放，验证巡检真实性 |
| 发现 | 定位打标 | 问题点位自动标注到园区地图 |
| 工单 | 就近匹配 | 空间计算最近可用处置人员 |
| 处置 | 导航指引 | GIS导航至问题点位 |
| 验收 | 现场确认 | 定位验证处置是否在正确点位完成 |
| 分析 | 热力图聚类 | DBSCAN/OPTICS聚类识别问题热区 |

## 六、场景间关联

三个场景并非孤立，而是共享数据、相互驱动：

- **日常巡检发现设备异常** → 自动触发设备检查场景
- **设备检查发现重大隐患** → 自动触发应急事件流程
- **应急事件处置完成后** → 数据进入分析引擎 → 优化日常巡检计划（如某区域水管爆裂后，提升该区域巡检频率）
- **分析引擎发现趋势** → 反向调整三个场景的计划（自优化闭环）

这种"场景联动+数据回流"的设计，使系统从"三个独立流程"进化为"一个有机整体"。
