# 04 场景详写① — 日常巡检全流程

> 基于03场景概述的七步闭环骨架，展开为完整的八步日常巡检流程（含"分支判断"步骤），标注每步的角色、AI嵌入、GIS嵌入、SLA时限，并描述轻微/严重两条分支路径及分析回流机制。

---

## 一、流程总览

日常巡检从计划编排到数据回流共8步，形成完整闭环：

```
①计划 → ②执行 → ③发现 → ④分支判断 → ⑤工单 → ⑥处置 → ⑦验收 → ⑧分析
                                                    ↑                                |
                                                    |_______ 数据回流自优化 _________|
```

> 分支判断（④）是03七步骨架中"发现"与"工单"之间的新增步骤，由AI辅助完成严重程度判断，决定走轻微当场闭环还是严重工单流程。

---

## 二、八步流程详解

### ① 计划 — 巡检计划编排与下发

| 维度 | 内容 |
|------|------|
| **角色** | 管理层（物业经理 王丽华）编排 → 巡检员（张建国/李明辉）接收 |
| **输入** | 历史巡检数据、园区布局图、问题热力图、上期分析报告 |
| **输出** | 当日巡检计划（含路线、点位、checklist、预计时长） |
| **触发方式** | 系统每日自动下发（晨会前完成），管理层可手动调整 |

**流程描述：**

1. 系统于每日06:00自动生成当日巡检计划，分配至对应巡检员的移动端
2. 巡检员打开APP后查看今日任务：巡检路线、点位清单、各点位checklist、预计完成时间
3. 管理层可在PC后台查看计划编排结果，并根据特殊情况（如重大活动、天气预警）手动调整路线或增删点位

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 预测分析 | 基于历史30天巡检数据，分析各区域问题发生频率，自动推荐巡检频次（如A区连续30天无问题→降频至每周2次；B区近期问题频发→提升至每日2次） |
| 路线优化 | 基于点位分布和历史巡检耗时，计算最优巡检路线，缩短总巡检时长 |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| 覆盖分析 | 检测巡检盲区——将巡检路线与园区全域范围叠加分析，若存在未覆盖区域，系统告警提示管理层补充点位 |
| 空间可视化 | 园区地图上展示巡检路线、点位分布，支持管理层直观审视计划合理性 |

**SLA：** 计划于每日06:00前完成编排下发；管理层调整需在巡检开始前30min完成。

---

### ② 执行 — 现场巡检执行

| 维度 | 内容 |
|------|------|
| **角色** | 巡检员（张建国/李明辉） |
| **输入** | 当日巡检计划（路线、点位、checklist） |
| **输出** | 巡检轨迹、各点位checklist完成记录、现场照片/视频 |
| **触发方式** | 巡检员到达首个点位后点击"开始巡检" |

**流程描述：**

1. 巡检员打开APP，点击"开始巡检"，系统自动进入导航模式
2. 按导航指引逐点巡检：到达点位 → 完成checklist（逐项勾选） → 拍照留证 → 点击"完成" → 导航至下一站
3. 巡检过程中实时记录GPS轨迹，管理层可在PC端查看巡检员实时位置
4. 系统显示巡检进度："已完成12/25个点位，剩余预计1h20min"

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 实时预警 | 若巡检员连续跳过2个以上checklist项（快速点击"完成"但未逐项检查），系统弹出提醒："请逐项完成检查"；若巡检员偏离路线超过50m，系统提醒回归路线 |
| 异常区域提示 | 接近历史高频问题区域时，系统主动提示："该区域近7天出现3次照明故障，请重点关注" |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| 轨迹记录 | GPS每10秒采样，记录巡检轨迹；管理层可在PC端回放巡检路线，验证是否真实到达每个点位 |
| 实时定位 | 巡检员位置实时展示在管理层后台，便于协调和调度 |
| 路线导航 | GIS内嵌导航，引导巡检员按最优路线前往下一个点位 |

**SLA：** 整条巡检路线需在计划时长内完成（如2h）；超时未完成系统自动告警管理层。

---

### ③ 发现 — 问题发现与上报

| 维度 | 内容 |
|------|------|
| **角色** | 巡检员（张建国/李明辉） |
| **输入** | 现场发现的异常/问题 |
| **输出** | 问题上报记录（含照片、语音、文字描述、定位、初步分类） |
| **触发方式** | 巡检员在巡检过程中发现问题，点击"上报问题" |

**流程描述：**

1. 巡检员发现异常（如消防通道堆放杂物），点击APP"上报问题"按钮
2. 拍照（支持多张） → 语音描述（系统自动转文字） → 选择问题类型（下拉菜单） → 提交
3. 系统自动附加：GPS定位、时间戳、巡检员信息、所在点位编号
4. 提交后系统即时反馈："问题已上报，编号INS-20260427-003，等待分类处理"

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 图像识别 | 拍照后AI自动识别问题类型（如"消防通道堆物"、"墙面破损"、"照明故障"），识别置信度≥85%时自动填充问题类型字段，<85%时由巡检员手动选择 |
| NLP结构化 | 语音转文字后，NLP自动提取关键信息：问题类型、严重程度关键词（如"堵塞"、"异味"、"异响"）、影响范围，结构化为工单预填字段 |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| 定位打标 | 问题点位自动标注到园区GIS地图，生成问题标记图层；若GPS精度不足（如室内），巡检员可手动拖拽标记校准位置 |
| 空间关联 | 自动关联该点位所属区域、楼栋、楼层，便于后续统计分析 |

**SLA：** 从发现问题到完成上报≤3min（拍照+语音+提交）。

---

### ④ 分支判断 — 严重程度判断与路径选择

| 维度 | 内容 |
|------|------|
| **角色** | 系统自动判断（AI辅助） + 巡检员确认 |
| **输入** | 步骤③的上报记录（含AI识别结果） |
| **输出** | 分支判定结果：轻微（当场闭环）或严重（进入工单流程） |
| **触发方式** | 步骤③提交后自动触发 |

**流程描述：**

1. 问题上报后，系统根据AI识别结果自动判定严重等级：
   - **轻微（P3）**：环境卫生类（纸屑、污渍、绿化落叶）、标识类（指示牌轻微歪斜）、物品归位类（椅子未归位）
   - **严重（P0-P2）**：安全隐患（消防通道堵塞、电气裸露）、设备故障（照明损坏、空调异常）、结构损坏（墙面裂缝、地砖破损）
2. 系统在APP上展示判定结果及建议路径，巡检员确认或手动修正
3. 确认后进入对应分支流程

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 严重程度预测 | 基于图像识别结果 + 问题类型 + 历史数据，AI自动判定严重等级（P0/P1/P2/P3），准确率目标≥90% |
| 分支引导 | 向巡检员展示判定依据（如"识别为地面纸屑，属轻微环境问题，建议当场处置"），帮助新手巡检员（李明辉）正确决策 |

**GIS嵌入：** 无独立GIS嵌入，复用步骤③的定位数据。

**SLA：** 分支判断≤30秒（系统自动完成，巡检员一键确认）。

---

### ⑤-轻 当场处置 — 轻微问题当场闭环

> 适用场景：地面有纸屑、桌面需擦拭、椅子需归位、标识牌轻微歪斜等。

| 维度 | 内容 |
|------|------|
| **角色** | 巡检员（张建国/李明辉）自行处置 |
| **输入** | 步骤④的轻微问题判定 |
| **输出** | 处置完成记录（处置后照片+时间戳） |
| **触发方式** | 步骤④判定为轻微后自动进入 |

**流程描述：**

1. 系统提示："轻微问题，建议当场处置"
2. 巡检员现场处理（如捡起纸屑、扶正标识牌）
3. 处置完成后拍照（"处置后"照片）
4. 系统自动比对处置前后照片，确认问题已解决
5. 该问题标记为"已闭环"，无需进入工单流程
6. 数据同步至分析引擎

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 图像比对 | 处置前后照片AI对比，自动判断问题是否有效解决（如"纸屑已清除"、"标识牌已恢复"）；若判定未解决，提示巡检员补充处置 |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| 定位验证 | 验证处置地点与问题上报地点是否一致（距离≤20m），防止虚假处置 |

**SLA：** 从发现到当场闭环≤10min。

---

### ⑤-严 工单创建与分派 — 严重问题进入工单流程

> 适用场景：设备故障、安全隐患、结构损坏等需专业人员处理的问题。

| 维度 | 内容 |
|------|------|
| **角色** | 系统自动创建工单 → 管理层（王丽华）确认分派 → 处置人接单 |
| **输入** | 步骤④的严重问题判定 + 步骤③的上报记录 |
| **输出** | 已分派工单（含处置人、SLA时限） |
| **触发方式** | 步骤④判定为严重后自动进入 |

**流程描述：**

1. 系统自动创建工单，预填字段（问题类型、照片、语音文字、定位、严重等级）均由前序步骤自动带入，无需巡检员重复录入
2. AI推荐处置人：基于技能匹配（消防问题→消防维保）+ 空间就近（GIS计算距问题点位最近）+ 当前负载（未完成工单数最少）
3. 管理层在PC后台确认或调整AI推荐的处置人，一键分派
4. 处置人收到APP推送通知，含工单详情、问题定位、导航入口

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 智能推荐 | 综合技能标签、当前GPS位置、历史处置效率、当前工单负载，推荐最优处置人（Top3候选） |
| 工单结构化 | NLP将巡检员的语音/文字描述自动结构化为标准工单字段（问题类型、位置描述、影响范围、紧急程度），减少管理层审核工作量 |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| 就近匹配 | 基于处置人实时GPS位置，计算距问题点位的步行/驾车距离，作为推荐权重之一 |
| 空间可视化 | 管理层在PC后台GIS地图上查看：问题点位+推荐处置人位置，直观判断分派合理性 |

**SLA：**
- 从问题上报到工单创建：≤1min（自动完成）
- 从工单创建到管理层确认分派：≤30min
- P1工单：处置人需在30min内到场
- P2工单：处置人需在2h内到场

---

### ⑥ 处置 — 问题处置与反馈

| 维度 | 内容 |
|------|------|
| **角色** | 处置人（维修队/外包/巡检员） |
| **输入** | 已分派工单 |
| **输出** | 处置完成记录（处置描述、处置后照片、耗时） |
| **触发方式** | 处置人到场后点击"开始处置" |

**流程描述：**

1. 处置人收到工单通知，点击"导航前往"，GIS引导至问题点位
2. 到达后点击"开始处置"，系统记录到场时间（验证SLA）
3. 处置过程中可查看AI推荐的处置方案知识库（如"消防通道堆物：清理杂物→恢复通道畅通→拍照确认"）
4. 处置完成后填写处置描述 + 上传处置后照片 → 提交

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 处置方案推荐 | 基于问题类型匹配历史相似工单的处置方案，供处置人参考（如"同类消防通道堆物问题，历史平均处置时间15min，常用方案：清理杂物+张贴警示标识"） |
| 到场时间验证 | 自动对比"分派时间→到场时间"与SLA要求，超时则触发升级 |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| 导航指引 | 从处置人当前位置导航至问题点位，支持步行/驾车模式 |
| 位置验证 | 到场时验证处置人GPS与问题点位距离≤50m，防止未到场虚报处置 |

**SLA：**
- P1工单：从分派到处置完成≤2h
- P2工单：从分派到处置完成≤24h
- 超时升级规则：超时30min→通知主管；超时1h→通知经理

### ⑦ 验收 — 处置结果验收

| 维度 | 内容 |
|------|------|
| **角色** | 巡检员（下次巡检时复核）或管理层（远程验收照片） |
| **输入** | 处置完成记录 |
| **输出** | 验收结果（通过/不通过） |
| **触发方式** | 处置提交后自动触发验收流程 |

**流程描述：**

1. 处置完成后，系统自动推送验收任务至对应巡检员或管理层
2. **巡检员验收**：下次巡检经过该点位时，现场复核并拍照 → 对比处置前后照片 → 确认问题是否彻底解决
3. **管理层验收**：远程查看处置后照片 + AI图像比对结果，快速决策
4. 验收通过 → 工单闭环；验收不通过 → 工单退回处置人，标注不通过原因，重新处置

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 图像比对 | 自动对比"上报时照片"与"处置后照片"，判断问题是否有效解决（如"堆物已清除"、"设备已修复"），给出置信度评分≥85%判定通过 |
| 质量评分 | 基于图像比对结果+处置描述完整性+到场时效，综合评分处置质量（1-5分），纳入处置人绩效参考 |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| 现场确认 | 巡检员验收时，验证GPS定位是否在问题点位附近（距离≤20m），确保现场验收真实性 |

**SLA：**
- 巡检员验收：处置完成后≤24h内完成
- 管理层验收：处置完成后≤4h内完成（远程快速验收）
- 验收不通过退回重处置：SLA重新计算

### ⑧ 分析 — 数据分析与决策优化

| 维度 | 内容 |
|------|------|
| **角色** | 管理层（王丽华/赵明远）查看 → 系统自动产出分析报告 |
| **输入** | 全流程数据（计划、轨迹、问题、工单、处置、验收） |
| **输出** | 日/周/月分析报告、优化建议、巡检计划调整方案 |
| **触发方式** | 每日自动生成日报；每周一生成周报；每月1日生成月报 |

**流程描述：**

1. 系统每日22:00自动汇总当日巡检数据，生成日报
2. 日报核心指标：巡检完成率、问题发现数、工单闭环率、SLA达成率、平均处置时长
3. 管理层在PC后台查看可视化报表，支持按区域/时间/问题类型下钻
4. 系统自动产出优化建议，管理层确认后进入计划调整流程

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 趋势预测 | 基于历史数据预测未来7天各区域问题发生概率，支持预防性巡检资源调配 |
| 异常检测 | 识别异常模式——如某区域问题数量突增50%、某巡检员连续3天漏检率>10%，自动告警 |
| 优化建议 | 自动生成巡检计划优化建议（如"B区3楼近30天消防通道堆物5次，建议将巡检频次从每日1次提升至每日2次"） |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| 热力图聚类 | 基于DBSCAN/OPTICS算法对问题点位进行空间聚类，生成问题热力图，识别"问题高发区域" |
| 覆盖率分析 | 叠加巡检轨迹与园区全域，计算空间覆盖率；识别持续未覆盖的"巡检盲区" |

**SLA：** 日报于当日22:30前自动生成；管理层查看无时限。

## 三、数据回流闭环

分析（⑧）产出的优化建议，经管理层确认后回流至计划（①），驱动巡检计划自优化：

```
⑧分析 → 优化建议 → 管理层确认 → ①计划自动更新 → ②执行调整后的巡检
```

### 回流触发的计划调整类型

| 调整类型 | 触发条件 | 示例 |
|---------|---------|------|
| **频次调整** | 某区域连续N天无问题（降频）或问题频发（升频） | A区连续30天无问题→降频至每周2次；B区7天内5次堆物→升频至每日2次 |
| **路线调整** | 热力图发现新增问题高发区或覆盖盲区 | GIS聚类发现D区新增隐患→将D区纳入巡检路线 |
| **点位调整** | 某点位长期无问题（移除）或新发现问题区域（新增） | E区标识牌点位连续60天无异常→移除；F区绿化带新发现问题→新增点位 |
| **人员调整** | 巡检员漏检率高或巡检效率低 | 李明辉连续3天漏检率>10%→安排老员工张建国带教1周 |
