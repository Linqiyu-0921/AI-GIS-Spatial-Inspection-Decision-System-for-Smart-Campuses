# 05 场景详写② — 设备专项检查全流程

> 基于03场景概述的七步闭环骨架，展开为完整的七步设备专项检查流程。与日常巡检的核心差异：检查清单驱动（非路线驱动）、BIM室内定位（非GPS室外定位）、AI外观识别+OCR（非通用图像识别）、技术验收（非简单复核）。

---

## 一、流程总览

设备专项检查从计划编排到数据分析共7步，形成完整闭环：

```
①计划 → ②执行 → ③发现 → ④工单 → ⑤处置 → ⑥技术验收 → ⑦分析
                  ↑                                              |
                  |___________ 数据回流自优化 ___________|
```

### 与日常巡检的核心差异

| 维度 | 日常巡检 | 设备专项检查 |
|------|---------|-------------|
| **驱动方式** | 路线驱动（按地理路线逐点巡检） | 检查清单驱动（按设备checklist逐项检查） |
| **定位方式** | GPS/GIS室外定位 | BIM/室内定位（设备在楼层内部） |
| **AI能力** | 通用图像识别+NLP结构化 | 外观识别（锈蚀/泄漏/变形）+OCR表盘读数 |
| **验收方式** | 巡检员复核或管理层远程验收照片 | 技术主管专业验收+功能测试 |
| **频率** | 每日1-2次 | 按设备类型（周检/月检/季检/半年检） |
| **处置复杂度** | 简单维修（保洁/更换灯泡） | 专业维保单位（电梯/消防/电力） |
| **闭环分支** | 轻微当场闭环/严重走工单 | 全部走工单（设备问题无轻微之说） |

## 二、设备检查清单模板

### 2.1 电梯检查清单（月检）

| 序号 | 检查项目 | 检查标准 | 检查方法 | AI辅助 |
|------|---------|---------|---------|--------|
| 1 | 运行平稳性 | 启停平稳，无异常抖动或异响 | 乘坐体验+录音 | 声音识别：异常频率分析 |
| 2 | 门机系统 | 开关门顺畅，防夹功能有效 | 反复开关测试 | 视频识别：门缝异物检测 |
| 3 | 安全装置 | 限速器、安全钳、缓冲器正常 | 目视+功能测试 | 外观识别：锈蚀/变形 |
| 4 | 通讯系统 | 轿厢内紧急通话清晰 | 实际通话测试 | — |
| 5 | 照明通风 | 轿厢照明充足，通风正常 | 目视+体感 | 图像识别：照度判断 |
| 6 | 显示系统 | 楼层显示准确，到站提示正常 | 目视 | OCR：显示屏读数校验 |
| 7 | 钢丝绳 | 无断丝、磨损、锈蚀 | 目视+手触 | 外观识别：断丝/锈蚀 |
| 8 | 导轨润滑 | 润滑良好，无干磨异响 | 目视+声音 | 声音识别：干磨频率 |

### 2.2 配电房检查清单（周检）

| 序号 | 检查项目 | 检查标准 | 检查方法 | AI辅助 |
|------|---------|---------|---------|--------|
| 1 | 变压器温度 | 绕温≤85℃，油温≤65℃ | 温控器读数 | OCR：温度表盘识别 |
| 2 | 开关柜状态 | 分合闸指示正确，无异味 | 目视+嗅觉 | 外观识别：指示灯状态 |
| 3 | 接地系统 | 接地线连接牢固，无锈蚀 | 目视+手动检查 | 外观识别：锈蚀/松动 |
| 4 | 通风散热 | 风机运转正常，温度≤40℃ | 目视+温度计 | OCR：温度读数 |
| 5 | 消防设施 | 灭火器在有效期内，气体灭火正常 | 目视 | OCR：灭火器有效期 |
| 6 | 电缆桥架 | 无发热变色，无杂物堆放 | 目视+红外测温 | 图像识别：变色/杂物 |
| 7 | 防潮防鼠 | 封堵完好，无鼠害痕迹 | 目视 | 图像识别：孔洞/鼠迹 |
| 8 | 运行声音 | 无异常放电声或振动 | 听诊+录音 | 声音识别：放电频率 |

### 2.3 消防泵房检查清单（季检）

| 序号 | 检查项目 | 检查标准 | 检查方法 | AI辅助 |
|------|---------|---------|---------|--------|
| 1 | 水泵运行 | 启动正常，运行平稳无异响 | 手动/自动启动测试 | 声音识别+振动分析 |
| 2 | 阀门状态 | 阀门开启到位，无泄漏 | 目视+手动检查 | 外观识别：泄漏/锈蚀 |
| 3 | 压力表读数 | 出口压力≥0.5MPa，稳压罐正常 | 目视 | OCR：压力表盘识别 |
| 4 | 管道连接 | 法兰连接无渗漏，支架牢固 | 目视 | 外观识别：渗漏/锈蚀 |
| 5 | 应急电源 | 双电源切换正常，蓄电池电压≥12V | 功能测试 | OCR：电压表读数 |
| 6 | 控制柜 | 指示灯正常，手/自动切换正常 | 目视+功能测试 | 图像识别：指示灯状态 |
| 7 | 水箱水位 | 水位≥2/3，无渗漏 | 目视 | 图像识别：水位线 |
| 8 | 管道标识 | 标识清晰，流向箭头正确 | 目视 | 图像识别：标识完整性 |

## 三、七步流程详解

### ① 计划 — 设备检查计划编排与下发

| 维度 | 内容 |
|------|------|
| **角色** | 技术主管（赵明远）编排 → 技术员（张建国）接收 |
| **输入** | 设备台账、维保规范、历史检查数据、设备运行状态 |
| **输出** | 设备检查计划（含设备清单、检查项目、检查频率、预计时长） |
| **触发方式** | 系统按设备维保规范自动排期；设备告警触发临时检查 |

**流程描述：**

1. 系统根据设备台账和维保规范，自动生成周期性检查计划：
   - 电梯：每月1次全面检查
   - 配电房：每周1次巡检，每月1次深度检查
   - 消防泵房：每季度1次全面检查
   - 空调系统：换季前（4月/10月）各1次检查
2. 技术员在移动端查看本周/本月待检设备清单，按优先级排序
3. 技术主管可在PC后台调整检查计划（如新增临时检查、调整检查顺序）

**设备检查频率规范：**

| 设备类型 | 检查频率 | 检查深度 | 检查人员 |
|---------|---------|---------|---------|
| 电梯 | 月检 | 全面检查（8项） | 技术员+电梯维保 |
| 配电房 | 周检+月检 | 周检：外观+温度；月检：全面 | 技术员 |
| 消防泵房 | 季检 | 全面检查（8项） | 技术员+消防维保 |
| 空调系统 | 半年检 | 换季前全面检查 | 技术员+空调维保 |
| 照明系统 | 月检 | 区域抽检 | 巡检员 |
| 给排水 | 月检 | 管道+水泵检查 | 技术员 |

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 预测分析 | 基于设备历史故障数据，预测高风险设备，优先安排检查（如C区电梯近3个月故障率上升→提升至每周检查） |
| 智能排期 | 综合设备重要性、历史故障率、维保单位档期，自动生成最优检查排期 |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| BIM空间可视化 | 在BIM模型上展示待检设备位置，技术主管直观审视检查覆盖范围 |
| 设备分布分析 | 识别检查盲区——某楼层设备长期未检查，系统自动告警 |

**SLA：** 计划于检查日前1天完成编排下发；设备告警触发的临时检查≤2h内响应。

### ② 执行 — 按检查清单逐项检查

| 维度 | 内容 |
|------|------|
| **角色** | 技术员（张建国） |
| **输入** | 设备检查计划+设备检查清单模板 |
| **输出** | 检查记录（含各检查项结果、现场照片、设备参数读数） |
| **触发方式** | 技术员到达设备点位后点击"开始检查" |

**流程描述：**

1. 技术员打开APP，查看待检设备清单，点击"开始检查"
2. 系统展示该设备类型的检查清单模板（如电梯检查清单8项）
3. 技术员按清单逐项检查：
   - **外观类项目**：拍照留证（如钢丝绳、阀门、管道）
   - **参数类项目**：拍摄表盘读数（如温度表、压力表、电压表），系统自动OCR识别数值
   - **功能类项目**：执行功能测试后记录结果（如电梯开关门测试、水泵启动测试）
4. 每完成一项，点击"完成"并填写备注（可选）
5. 全部检查项完成后，点击"提交检查记录"

**BIM/室内定位嵌入：**

| 定位能力 | 具体作用 |
|---------|---------|
| BIM楼层定位 | 技术员进入楼层后，BIM自动定位到当前楼层，展示该楼层所有待检设备位置 |
| 设备导航 | 在BIM模型上点击目标设备，系统规划室内导航路径（如"左转→第三个门→配电房"） |
| 到位验证 | 技术员到达设备附近（蓝牙信标/二维码扫码），系统验证已到位后才允许开始检查 |

**SLA：** 单台设备检查需在计划时长内完成（如电梯检查30min，配电房巡检20min）。

### ③ 发现 — 设备问题发现与AI辅助判断

| 维度 | 内容 |
|------|------|
| **角色** | 技术员（张建国）发现 → AI辅助判断严重程度 |
| **输入** | 检查过程中发现的异常（外观异常、参数超标、功能异常） |
| **输出** | 设备问题记录（含异常照片、OCR读数、AI判断结果、严重等级） |
| **触发方式** | 技术员在检查清单中标记某项"异常" |

**流程描述：**

1. 技术员在检查过程中发现异常（如电梯运行异响、配电房温度偏高、消防管道渗漏）
2. 在检查清单中将该项标记为"异常"，系统弹出异常上报界面：
   - 拍照（支持多角度多张）
   - 录音（记录异常声音）
   - 填写异常描述（或语音转文字）
3. AI自动分析异常数据：
   - **外观照片**：识别锈蚀、泄漏、变形、裂纹等缺陷类型
   - **表盘读数**：OCR识别数值，对比标准范围，判断是否超标
   - **声音录音**：分析异常频率，判断异响类型
4. AI综合判断严重等级并给出建议：
   - **紧急（P0）**：影响安全运行，需立即停机维修（如电梯安全装置失效、配电房温度超标）
   - **严重（P1）**：影响正常使用，需尽快维修（如水泵异响、阀门泄漏）
   - **一般（P2）**：不影响运行，需计划维修（如表面锈蚀、标识脱落）
5. 技术员确认AI判断结果，提交异常记录

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| **外观识别** | 基于设备照片自动识别缺陷类型：锈蚀（表面氧化变色）、泄漏（液体/气体痕迹）、变形（结构形变）、裂纹（表面开裂）、磨损（表面损耗） |
| **OCR表盘识别** | 识别设备表盘读数：温度表（绕温/油温）、压力表（出水压力/气压）、电压表（蓄电池电压）、电流表（运行电流），自动与标准范围比对，超标则告警 |
| **声音分析** | 录音后分析频谱，识别异常声音类型：异响（机械摩擦）、放电声（电气故障）、振动异常（不平衡/松动） |
| **综合判断** | 综合外观+参数+声音三项数据，AI给出设备健康度评分（0-100分）和严重等级建议 |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| BIM定位打标 | 异常设备在BIM模型上标记，管理层可直观查看异常设备分布 |
| 空间关联 | 自动关联设备所属楼层、区域、所属系统（电梯/电力/消防），便于统计分析 |

**SLA：** 异常发现到完成上报≤5min（拍照+OCR+AI判断+提交）。

### ④ 工单 — 设备维修工单创建与分派

| 维度 | 内容 |
|------|------|
| **角色** | 系统自动创建工单 → 技术主管（赵明远）审核分派 → 维保单位接单 |
| **输入** | 步骤③的异常记录（含AI判断结果） |
| **输出** | 已分派的设备维修工单（含维保单位、SLA时限） |
| **触发方式** | 步骤③提交后自动触发 |

**流程描述：**

1. 系统自动创建设备维修工单，预填字段由前序步骤自动带入：
   - 设备信息（设备编号、名称、位置、所属系统）
   - 异常描述（照片、OCR读数、AI判断结果、严重等级）
   - 建议维修方案（基于历史相似工单推荐）
2. 技术主管在PC后台审核工单：
   - 确认AI判断的严重等级是否准确
   - 确认是否需要外部维保单位（如电梯维保、消防维保）
   - 设定SLA时限（紧急48h/一般7d）
3. 分派至对应维保单位或内部维修队
4. 维保单位收到APP推送，含工单详情、设备位置、BIM导航入口

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 智能推荐 | 基于设备类型+故障类型+历史合作记录，推荐最优维保单位（Top3候选） |
| 方案推荐 | 匹配历史相似工单的维修方案，供技术主管参考（如"电梯异响-导轨问题：历史平均维修耗时4h，常用方案：导轨润滑+调整"） |
| SLA建议 | 基于故障严重程度和设备重要性，自动建议SLA时限 |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| BIM设备定位 | 工单自动关联设备在BIM模型中的精确位置（楼层+房间+具体位置），维保单位可直接导航 |
| 就近匹配 | 基于维保单位服务范围和当前位置，推荐最近可用单位 |

**SLA：**
- 从异常上报到工单创建：≤1min（自动完成）
- 从工单创建到技术主管审核：隐患审核≤24h
- 紧急维修：P0/P1工单≤48h完成
- 一般维修：P2工单≤7d完成

### ⑤ 处置 — 设备维修处置

| 维度 | 内容 |
|------|------|
| **角色** | 维保单位/内部维修队 |
| **输入** | 已分派的设备维修工单 |
| **输出** | 维修完成记录（维修报告、更换部件清单、维修后照片、测试数据） |
| **触发方式** | 维保单位到场后点击"开始维修" |

**流程描述：**

1. 维保单位收到工单通知，通过BIM导航至设备位置
2. 到达后点击"开始维修"，系统记录到场时间（验证SLA）
3. 维修过程中：
   - 查看AI推荐的维修方案和历史相似案例
   - 拍照记录维修过程（更换部件、维修操作）
   - 填写维修报告（故障原因、维修措施、更换部件）
4. 维修完成后进行功能测试：
   - 电梯：试运行2个来回，确认运行正常
   - 配电房：测量各项参数，确认在标准范围
   - 消防泵房：启动测试，确认出水压力达标
5. 上传维修报告+测试数据+维修后照片 → 提交

**BIM/室内定位嵌入：**

| 定位能力 | 具体作用 |
|---------|---------|
| BIM室内导航 | 从维保单位进入大楼开始，BIM提供逐层导航：大厅→电梯→目标楼层→设备房 |
| 到位验证 | 维保单位到达设备附近后，通过蓝牙信标/二维码扫码验证已到位，防止未到场虚报 |
| 维修区域标注 | 在BIM模型上标注正在维修的设备，提醒其他人员注意安全 |

**SLA：**
- P0/P1工单：从分派到维修完成≤48h
- P2工单：从分派到维修完成≤7d
- 超时升级规则：超时24h→通知技术主管；超时48h→通知物业经理

### ⑥ 技术验收 — 专业技术人员验收

| 维度 | 内容 |
|------|------|
| **角色** | 技术主管（赵明远）或指定技术专家 |
| **输入** | 维修完成记录（维修报告、测试数据、维修后照片） |
| **输出** | 验收结果（通过/不通过）+ 验收报告 |
| **触发方式** | 维修提交后自动触发验收流程 |

**流程描述：**

1. 维修完成后，系统自动推送验收任务至技术主管
2. 技术主管在PC后台审核维修报告：
   - 检查维修措施是否针对故障根因
   - 检查更换部件是否符合规格
   - 检查测试数据是否达标
3. **远程审核**：查看维修照片+测试数据+AI图像比对结果，快速判断
4. **现场验收**（必要时）：
   - 技术主管到达设备现场
   - 独立进行功能测试（不依赖维保单位的测试结果）
   - 拍照记录验收状态
   - 在APP上填写验收报告
5. 验收通过 → 工单闭环；验收不通过 → 工单退回维保单位，标注不通过原因，重新维修

**与日常巡检验收的核心差异：**

| 维度 | 日常巡检验收 | 设备技术验收 |
|------|-------------|-------------|
| **验收人员** | 巡检员或管理层 | 技术主管或指定技术专家 |
| **验收内容** | 照片比对（处置前后） | 功能测试+参数验证+照片比对 |
| **验收标准** | 问题是否消失 | 设备功能是否恢复正常+参数是否达标 |
| **验收深度** | 表面确认 | 深度技术验证（可能需要现场） |

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 图像比对 | 自动对比"上报时照片"与"维修后照片"，判断外观缺陷是否修复（如锈蚀已处理、泄漏已修复） |
| 参数验证 | OCR读取维修后测试数据，自动比对标准范围，判断参数是否达标 |
| 质量评分 | 综合图像比对+参数验证+维修报告完整性，给出维修质量评分（1-5分） |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| BIM现场定位 | 技术主管现场验收时，BIM定位验证是否在设备附近，确保现场验收真实性 |
| 验收状态标注 | 验收通过的设备在BIM模型上恢复正常状态标识 |

**SLA：**
- 远程审核：维修完成后≤24h内完成
- 现场验收：维修完成后≤48h内完成
- 验收不通过退回重维修：SLA重新计算

### ⑦ 分析 — 设备健康度分析与决策优化

| 维度 | 内容 |
|------|------|
| **角色** | 技术主管（赵明远）查看 → 系统自动产出分析报告 |
| **输入** | 全流程数据（检查记录、异常数据、工单、维修、验收） |
| **输出** | 设备健康度报告、维保建议、设备更新建议 |
| **触发方式** | 每月自动生成设备健康度报告；重大故障触发专项分析 |

**流程描述：**

1. 系统每月1日自动生成设备健康度月报
2. 月报核心指标：
   - 设备完好率（正常运行设备数/总设备数）
   - 故障率（故障设备数/总设备数）
   - 平均故障修复时间（MTTR）
   - 维修工单闭环率
   - SLA达成率
3. 技术主管在PC后台查看可视化报表，支持按设备类型/楼层/系统下钻
4. 系统自动产出决策建议：
   - 预防性维保建议（基于设备劣化趋势）
   - 设备更新建议（基于故障率上升趋势）
   - 维保单位绩效评估

**AI嵌入：**

| AI能力 | 具体作用 |
|--------|---------|
| 设备劣化预测 | 基于历史检查数据（OCR读数趋势、外观缺陷频率），预测设备劣化曲线，提前预警（如"电梯钢丝绳磨损速度加快，预计6个月后需更换"） |
| 异常模式识别 | 识别设备异常模式：某类设备集中故障（如B区电梯近3个月故障率上升40%）、某维保单位维修质量下降 |
| 优化建议 | 自动生成维保策略优化建议（如"消防泵房检查频率从季检提升至月检"、"建议更换XX维保单位"） |

**GIS嵌入：**

| GIS能力 | 具体作用 |
|---------|---------|
| BIM热力图 | 在BIM模型上生成设备故障热力图，识别故障高发楼层/区域 |
| 设备生命周期 | 在BIM模型上展示设备年龄、维修历史、健康度评分，支持设备更新决策 |

**SLA：** 月报于每月3日前自动生成；重大故障专项分析≤72h。

## 四、数据回流闭环

分析（⑦）产出的优化建议，经技术主管确认后回流至计划（①），驱动设备检查计划自优化：

```
⑦分析 → 优化建议 → 技术主管确认 → ①计划自动更新 → ②执行调整后的检查
```

### 回流触发的计划调整类型

| 调整类型 | 触发条件 | 示例 |
|---------|---------|------|
| **频率调整** | 设备故障率上升（升频）或持续稳定（降频） | C区电梯故障率上升→月检改为双周检；A区配电房连续6个月无异常→周检改为双周检 |
| **检查深度调整** | 某类设备频繁出现同类故障 | 消防泵房阀门频繁泄漏→检查清单增加"阀门密封圈"专项检查 |
| **设备更新** | 设备健康度持续低于阈值 | 某电梯健康度评分连续3个月<60分→建议大修或更换 |
| **维保单位调整** | 维保单位维修质量评分下降 | XX维保单位连续2次验收不通过→更换维保单位 |

## 七、与日常巡检的差异化对比总表

| 维度 | 日常巡检 | 设备专项检查 |
|------|---------|-------------|
| **核心驱动** | 路线驱动（按地理路线） | 检查清单驱动（按设备checklist） |
| **检查对象** | 公共区域环境+安全 | 专业设备（电梯/电力/消防） |
| **定位技术** | GPS/GIS室外定位 | BIM/室内定位（蓝牙信标/二维码） |
| **AI核心能力** | 通用图像识别+NLP | 外观识别+OCR表盘+声音分析 |
| **问题类型** | 环境卫生+安全隐患+简单维修 | 设备故障+参数异常+功能失效 |
| **闭环分支** | 轻微当场闭环/严重走工单 | 全部走工单（设备问题无轻微） |
| **处置人员** | 巡检员/保洁/简单维修 | 专业维保单位（电梯/消防/电力） |
| **验收标准** | 问题是否消失（表面确认） | 功能是否恢复+参数是否达标（技术验证） |
| **验收人员** | 巡检员或管理层 | 技术主管或技术专家 |
| **分析维度** | 问题热力图+巡检覆盖率 | 设备健康度+劣化趋势+维保绩效 |
| **频率** | 每日1-2次 | 按设备类型（周/月/季/半年） |
| **SLA** | 处置P1≤2h / P2≤24h | 隐患审核≤24h / 紧急≤48h / 一般≤7d |
