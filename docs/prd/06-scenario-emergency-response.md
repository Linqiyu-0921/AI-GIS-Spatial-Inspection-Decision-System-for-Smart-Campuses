# 06 应急事件全流程

> 定义应急事件的非线性、并行响应流程，实现IoT+人工双确认、指挥中心调度与态势地图实时追踪，确保突发事件快速、高效处置。

---

## 一、应急场景定义

应急事件是指园区内突然发生、需立即响应处置的紧急情况，包括但不限于：

| 事件类型 | 典型场景 | 危害等级 | 主要处置角色 |
|---------|---------|---------|------------|
| **火灾** | 电气短路、明火、烟雾 | P0 | 安保+消防+全员疏散 |
| **水管爆裂** | 主水管破裂、阀门失效 | P0 | 维修队+安保 |
| **电梯困人** | 电梯故障、停电 | P0 | 维保单位+安保 |
| **人员受伤** | 滑倒、高空坠物 | P0/P1 | 安保+医疗+管理层 |
| **设备重大故障** | 配电房异常、空调系统瘫痪 | P1 | 维修队+管理层 |

---

## 二、应急响应分级与SLA

### 2.1 应急独立SLA（区别于日常巡检）

| 阶段 | SLA时限 | 说明 |
|------|--------|------|
| **确认** | ≤3min | IoT+人工双确认事件真实性 |
| **到场** | ≤10min | 处置人员到达现场 |
| **响应** | ≤10min | 指挥中心启动预案、调度资源 |

> **注**：应急SLA独立于日常巡检SLA（日常巡检为P2/P3级，响应≤2h/24h），应急事件为P0/P1级，要求分钟级响应。

### 2.2 应急优先级定义

| 级别 | 定义 | 响应方式 | 升级规则 |
|------|------|---------|---------|
| **P0** | 重大安全事件，需立即响应 | 全员推送+指挥中心接管+并行响应 | 3min未确认→自动升级至最高级；连续2次不确认→强制升级 |
| **P1** | 严重隐患，需尽快处理 | 相关角色推送+指挥中心协调 | 10min未响应→升级至P0 |

## 三、应急全流程（七步闭环）

应急流程采用**非线性、并行响应**模式，与日常巡检的线性流程有本质区别。

```
触发 → 确认 → 研判调度 → 并行响应 → 态势追踪 → 解除 → 复盘
```

### 3.1 触发（事件发生）

| 触发方式 | 具体场景 | 系统响应 |
|---------|---------|---------|
| **IoT告警** | 烟雾传感器报警、水浸传感器触发、电梯故障信号 | 系统自动接收告警，立即启动应急流程 |
| **人工紧急上报** | 巡检员发现火情、员工报告水管爆裂 | 巡检员/员工通过App一键上报，系统立即启动应急流程 |
| **系统检测** | 视频AI识别到异常行为、设备参数超阈值 | 系统自动分析，触发应急流程 |

**触发规则**：
- 任何单一触发源均启动应急流程，不等待多源确认（确认环节并行进行）
- 触发后系统自动记录：时间、位置、触发源、初始描述

### 3.2 确认（IoT+人工双确认，≤3min）

确认环节采用**并行核查**机制，多源数据同时验证：

| 确认方式 | 执行者 | 动作 | 时限 |
|---------|--------|------|------|
| **IoT传感器核查** | 系统自动 | 读取相关传感器数据（温度、水位、烟雾浓度等） | ≤30s |
| **视频联动核查** | 系统自动 | 调取最近摄像头画面，AI分析是否匹配告警类型 | ≤1min |
| **就近人员现场确认** | 最近可用人员（巡检员/安保/员工） | 接收推送，前往现场确认，拍照/视频上传 | ≤3min |

**确认逻辑**：
- **双源确认**：IoT+视频任一确认即视为有效告警
- **人工确认**：就近人员确认为最终确认，如与IoT/视频不一致，以人工确认为准
- **超时处理**：3min内无任何确认 → 自动升级至最高级P0，全员推送

### 3.3 研判调度（指挥中心接管）

确认后，指挥中心立即接管，进行事件定级与资源调度：

| 步骤 | 执行者 | 动作 | 输出 |
|------|--------|------|------|
| **事件定级** | 指挥中心+AI | 基于事件类型、影响范围、危害程度确定P0/P1级别 | 事件级别、响应范围 |
| **GIS缓冲区分析** | 系统自动 | 以事件点为中心，分析影响半径（如火灾扩散范围、水管爆裂影响区域） | 影响范围、需疏散区域、可用资源分布 |
| **AI预案匹配** | AI引擎 | 匹配历史类似事件处置预案，推荐最佳响应方案 | 预案推荐、处置步骤、所需资源 |
| **资源调度** | 指挥中心 | 基于GIS就近分析，调度最近可用人员、设备、物资 | 调度指令、责任人、预计到场时间 |

**调度规则**：
- P0事件：指挥中心直接接管，所有角色立即响应
- P1事件：指挥中心协调，相关角色响应
- GIS缓冲区分析半径：火灾50m、水管爆裂30m、电梯困人10m（可配置）

### 3.4 并行响应（多角色同时执行）

应急响应采用**多线程并行**模式，各角色同时执行不同任务：

| 线程 | 角色 | 任务 | 与其他线程关系 |
|------|------|------|--------------|
| **线程1：现场处置** | 巡检员/维修队 | 到达现场，执行处置操作（灭火、关阀、急救等） | 主线程，其他线程配合 |
| **线程2：设备抢修** | 维保团队 | 设备故障抢修（电梯、水管、电气等） | 与线程1并行，技术处置 |
| **线程3：疏散引导** | 安保人员 | 疏散周边人员，设置警戒线，引导至安全区域 | 与线程1并行，安全保障 |
| **线程4：资源协调** | 管理层 | 协调外部资源（消防、医疗、供应商），信息上报 | 支持线程，资源保障 |

**并行规则**：
- 各线程独立执行，通过指挥中心协调
- 线程间信息实时同步（处置进展、资源需求、人员状态）
- 任一线程完成任务后，向指挥中心报告，等待指令

### 3.5 态势地图（指挥中心实时追踪）

指挥中心通过态势地图实时监控各线程状态：

| 监控维度 | 显示内容 | 更新频率 |
|---------|---------|---------|
| **事件位置** | 事件点位、影响范围（GIS缓冲区） | 实时 |
| **人员分布** | 各线程人员位置、移动轨迹 | 每10s |
| **资源状态** | 设备、物资使用情况 | 每30s |
| **处置进展** | 各线程任务完成度、关键节点 | 事件驱动 |
| **时间线** | 各阶段耗时、SLA达成情况 | 实时 |

**态势地图功能**：
- **GIS叠加**：事件点、影响范围、人员位置、资源分布多图层叠加
- **实时通信**：指挥中心与各线程语音/文字实时通信
- **预警提示**：SLA即将超时、资源不足、线程异常等自动预警
- **历史回放**：事件处置全过程可回放，用于复盘分析

### 3.6 解除（各线程逐一确认→恢复常态）

| 步骤 | 执行者 | 动作 | 确认标准 |
|------|--------|------|---------|
| **现场处置确认** | 巡检员/维修队 | 确认处置完成，拍照上传 | 问题已解决，现场安全 |
| **设备恢复确认** | 维保团队 | 确认设备恢复正常运行 | 设备参数正常，无异常 |
| **人员清点确认** | 安保人员 | 确认疏散人员已清点，无遗漏 | 人员全部安全，警戒解除 |
| **指挥中心确认** | 管理层 | 综合各线程报告，确认事件解除 | 所有线程确认完成，无遗留风险 |

**解除规则**：
- 所有线程逐一确认后，指挥中心方可宣布事件解除
- 解除后系统自动恢复常态巡检计划
- 解除信息全员推送，包括事件 summary、处置结果、后续注意事项

### 3.7 复盘（AI自动生成复盘报告+预案优化建议）

| 复盘内容 | 生成方式 | 输出 |
|---------|---------|------|
| **事件时间线** | 系统自动记录各节点时间 | 从触发到解除的完整时间线 |
| **SLA达成分析** | 系统自动计算各阶段耗时 | 确认、到场、响应SLA达成情况 |
| **资源使用分析** | 系统记录资源调度情况 | 人员、设备、物资使用效率 |
| **处置效果评估** | AI分析处置前后对比 | 处置效果评分、改进建议 |
| **预案优化建议** | AI对比历史类似事件 | 预案优化点、流程改进建议 |

**复盘输出**：
- 自动生成《应急事件复盘报告》，包含数据、分析、建议
- 自动更新应急预案库，优化AI预案匹配模型
- 自动调整相关区域巡检频率（如水管爆裂后提升该区域巡检频率）

## 四、并行响应机制详解

### 4.1 多线程并行模型

```
指挥中心（态势地图）
    ├── 线程1：现场处置（巡检员/维修队）
    ├── 线程2：设备抢修（维保团队）
    ├── 线程3：疏散引导（安保人员）
    └── 线程4：资源协调（管理层）
```

### 4.2 线程间协同规则

| 协同场景 | 处理方式 |
|---------|---------|
| **资源冲突** | 指挥中心统一调度，优先级高的线程优先使用资源 |
| **信息同步** | 各线程通过指挥中心中转，避免直接通信混乱 |
| **任务依赖** | 如疏散完成是现场处置的前提，指挥中心协调时序 |
| **异常处理** | 任一线程异常，指挥中心立即调整其他线程任务 |

### 4.3 与日常巡检的本质区别

| 维度 | 日常巡检 | 应急事件 |
|------|---------|---------|
| **流程模式** | 线性单线程 | 非线性并行多线程 |
| **响应机制** | 按计划执行，异步处置 | 立即响应，同步处置 |
| **SLA要求** | 小时级（2h/24h） | 分钟级（3min/10min） |
| **指挥模式** | 巡检员自主执行 | 指挥中心统一调度 |
| **GIS应用** | 路线规划、点位定位 | 缓冲区分析、资源调度、态势地图 |
| **AI应用** | 图像识别、NLP结构化 | 预案匹配、复盘分析、优化建议 |

## 六、与其他场景的联动

### 6.1 从其他场景触发应急

| 触发源 | 场景 | 联动方式 |
|--------|------|---------|
| **日常巡检发现重大隐患** | 日常巡检 → 应急事件 | 巡检员上报，系统自动触发应急流程 |
| **设备检查发现重大故障** | 设备检查 → 应急事件 | 设备参数超阈值，系统自动触发应急流程 |
| **IoT持续告警** | 设备监控 → 应急事件 | 设备连续异常，系统自动升级为应急事件 |

### 6.2 应急事件反向影响其他场景

| 影响方向 | 具体影响 |
|---------|---------|
| **应急 → 日常巡检** | 事件处置完成后，自动提升相关区域巡检频率 |
| **应急 → 设备检查** | 设备故障事件后，自动安排专项设备检查 |
| **应急 → 预案优化** | 复盘报告自动更新应急预案库，优化AI匹配模型 |

### 6.3 数据回流与自优化闭环

应急事件数据自动进入分析引擎，驱动系统自优化：
- **事件聚类分析**：识别高频事件区域，优化巡检路线
- **SLA达成分析**：识别响应瓶颈，优化调度策略
- **预案效果评估**：评估各预案处置效果，优化预案库
- **资源使用优化**：分析资源使用效率，优化资源布局

## 七、应急流程示例：水管爆裂

### 7.1 事件背景
B区3楼主水管爆裂，大量漏水，影响办公区域。

### 7.2 流程推演

| 阶段 | 时间 | 动作 | 负责方 |
|------|------|------|--------|
| **触发** | T+0s | 水浸传感器报警+巡检员发现漏水上报 | 系统+巡检员 |
| **确认** | T+30s | IoT读取水浸传感器数据，确认漏水 | 系统自动 |
| | T+1min | 视频联动，调取B区3楼摄像头，AI确认大量漏水 | 系统自动 |
| | T+2min | 就近巡检员到达现场，拍照确认水管爆裂 | 巡检员 |
| **研判调度** | T+3min | 指挥中心定级P0，启动应急响应 | 指挥中心 |
| | T+3.5min | GIS缓冲区分析：影响半径30m，涉及B区3-4楼 | 系统自动 |
| | T+4min | AI匹配预案：关闭上游阀门+维修+疏散+清理 | AI引擎 |
| | T+4.5min | 调度：维修队（最近2人）、安保（最近3人）、保洁（最近2人） | 指挥中心 |
| **并行响应** | T+5min | 线程1：维修队到达，关闭上游阀门 | 维修队 |
| | T+5min | 线程2：安保疏散B区3-4楼人员，设置警戒线 | 安保 |
| | T+5min | 线程3：管理层协调外部资源（如需） | 管理层 |
| | T+15min | 维修队完成临时修复，漏水停止 | 维修队 |
| | T+20min | 安保确认人员疏散完成，无遗漏 | 安保 |
| **态势追踪** | T+0~25min | 指挥中心通过态势地图实时监控各线程状态 | 指挥中心 |
| **解除** | T+25min | 现场处置确认、设备恢复确认、人员清点确认 | 各线程 |
| | T+26min | 指挥中心确认事件解除，恢复常态 | 指挥中心 |
| **复盘** | T+30min | AI自动生成复盘报告：从发现到解除耗时26min，SLA达成 | 系统自动 |

### 7.3 SLA达成情况

| 阶段 | SLA要求 | 实际耗时 | 达成情况 |
|------|--------|---------|---------|
| 确认 | ≤3min | 2min | ✅ 达成 |
| 到场 | ≤10min | 5min | ✅ 达成 |
| 响应 | ≤10min | 4.5min | ✅ 达成 |

## 八、关键设计决策

1. **非线性并行响应**：应急流程不是单线程串行，而是多角色多线程并行，通过指挥中心协调，确保响应速度最大化。

2. **IoT+人工双确认**：不依赖单一确认源，IoT传感器、视频AI、人工现场确认三重验证，确保告警真实性。

3. **指挥中心统一调度**：应急事件由指挥中心接管，避免多头指挥，确保资源调度高效有序。

4. **态势地图实时追踪**：指挥中心通过态势地图实时掌握各线程状态，及时发现异常、调整策略。

5. **独立SLA体系**：应急SLA与日常巡检SLA分离，确保应急事件分钟级响应。

6. **AI深度参与**：从预案匹配到复盘优化，AI贯穿应急全流程，提升响应效率和处置质量。

7. **数据驱动自优化**：应急事件数据自动回流分析引擎，驱动预案优化、巡检调整、资源布局优化。
