## 一、市场背景

### 1.1 智慧园区市场规模

[假设] 根据行业研究报告，中国智慧园区市场规模预计2026年突破3000亿元，年复合增长率约15%。其中，园区运维管理（含巡检、安防、设备管理）占整体投入的20%-25%，是仅次于基础设施建设的第二大细分领域。

办公商务园区（写字楼集群、CBD）区别于产业园区和制造业园区，具备以下特征：

| 特征 | 说明 |
|------|------|
| 空间形态 | 多栋高层/超高层写字楼 + 公共区域（广场、停车场、绿化带） |
| 管理对象 | 电梯、空调、消防、照明、门禁等楼宇设备为主 |
| 人员密度 | 高密度白领租户，对环境品质和安全敏感度高 |
| 物业模式 | 第三方物业托管为主，管理标准化程度参差不齐 |

### 1.2 办公园区巡检痛点

当前办公商务园区巡检管理普遍存在以下结构性问题：

| 痛点 | 现状 | 后果 |
|------|------|------|
| **巡检靠腿、记录靠纸** | 巡检员手持纸质工单逐点打卡，发现问题手写记录 | 信息滞后、描述模糊、无法追溯 |
| **发现问题≠解决问题** | 巡检记录与工单系统割裂，发现隐患后需人工转录、手动派单 | 问题遗漏、响应延迟、责任不清 |
| **巡了什么≠该巡什么** | 巡检路线和频率固定，不根据历史数据动态调整 | 高风险区域巡检不足，低风险区域重复浪费 |
| **数据沉睡** | 巡检数据仅用于存档，不做趋势分析和决策支持 | 管理层无法预判风险，决策依赖经验 |
| **竞品方案不适配** | 海康偏安防硬件、广联达偏施工场景、开源方案无AI/GIS | 办公园区缺乏针对性解决方案 |

## 二、产品愿景

**一句话定位：**

> 让每一次巡检都有据可依、每一个问题都闭环解决、每一条数据都驱动决策——从"记录巡了什么"升级为"决策该怎么巡"。

**差异化定位：**

本项目不做"又一个功能更全的园区管理系统"，而是聚焦于以 GIS 空间决策为核心引擎、以 AI 全流程嵌入为技术底座、以自优化闭环为进化机制的办公商务园区巡检决策系统。

## 三、核心目标

以下为 V1.0 版本的可量化目标：

| 编号 | 目标 | 量化指标 | 衡量方式 |
|------|------|---------|---------|
| G1 | 问题发现到工单创建的时间缩短 | 从人工转录的 [假设] 30min 缩短至 ≤2min（AI自动结构化+自动触发） | 工单创建时间戳 - 问题上报时间戳的均值 |
| G2 | 工单闭环率提升 | 从 [假设] 70% 提升至 ≥95%（SLA驱动+超时升级） | 验收通过的工单数 / 总工单数 |
| G3 | 巡检覆盖率提升 | 从 [假设] 60% 提升至 ≥90%（覆盖盲区检测+路线优化） | GIS覆盖分析：已巡点位 / 应巡点位 |
| G4 | 管理决策有数据支撑 | 100% 的巡检数据进入分析引擎，产出热力图、趋势报告 | 数据入库率 + 报告产出频率 |
| G5 | 系统易用性达标 | 巡检员端上手培训时间 ≤30min，管理层端核心操作 ≤3步 | 新用户任务完成率测试 |

## 四、目标用户

系统服务两类核心角色：

| 角色 | 使用端 | 核心诉求 |
|------|--------|---------|
| **巡检员** | 移动端（APP/H5） | 快速接收任务、高效上报问题、减少重复劳动 |
| **管理层** | PC后台（Web） | 掌控全局态势、追踪问题闭环、基于数据做决策 |

管理层进一步细分为：物业经理（日常运营）、技术主管（设备与工单）、园区负责人（战略决策）。详见 `02-user-personas.md`。

## 五、产品范围（V1.0）

### 包含

| 模块 | 范围 |
|------|------|
| 巡检任务管理 | 计划编排、任务下发、执行记录、完成确认 |
| 问题上报与工单 | 移动端上报（拍照+语音+文字）→ AI结构化 → 自动/手动创建工单 → 分派 → 处置 → 验收 |
| AI辅助识别 | 巡检现场图像识别（缺陷/隐患分类）、上报内容NLP结构化 |
| GIS空间可视化 | 园区地图底图、巡检轨迹、问题点位标注、热力图聚类 |
| SLA与升级 | 工单分级（P0-P3）、时限配置、超时自动升级与转派 |
| 数据看板 | 巡检完成率、工单闭环率、SLA达成率、问题热力图 |
| 移动巡检端 | 巡检员登录、任务列表、地图导航、问题上报、工单处置 |

### 不包含（V1.0 明确排除）

| 排除项 | 理由 |
|--------|------|
| 3D数字孪生/BIM | V1.0仅2D GIS，3D为后续版本 |
| IoT设备对接 | V1.0聚焦人工巡检，IoT联动为V2.0 |
| 多园区管理 | V1.0单园区验证，多园区横向扩展为后续版本 |
| 财务/合同管理 | 超出巡检决策系统边界 |
| 无人机/机器人巡检 | V1.0仅移动端人工巡检 |
| 第三方系统集成 | V1.0独立运行，开放API为V2.0 |

## 六、成功标准

V1.0上线后 [假设] 3个月内达成以下标准视为项目成功：

1. 至少1个办公商务园区完成试点部署
2. 巡检员端日活使用率 ≥80%
3. 工单平均闭环时间缩短 ≥50%（对比上线前基线）
4. 管理层周报/月报100%由系统数据自动生成
5. 用户满意度评分 ≥4.0/5.0（巡检员+管理层分别评估）
